# Compute — How IBM Apptio Costing Standard Addresses This TBM Capability

## The capability

Compute gives financial and operational transparency to the Compute Resource Tower across on-premises, private cloud, and public cloud environments, combining infrastructure, software, labor, and support to understand real cost, unit economics, and utilization of physical and virtual resources.

Four personas anchor this capability: IT Finance, Service Owners, Solution and Application Owners, and FinOps/CCoE. Four out-of-the-box use cases: Hybrid Compute TCO (compare compute cost between on-premises, private cloud, and public cloud), Server Utilization and Efficiency (analyze CPU and memory utilization to identify underused capacity), Hybrid Infrastructure Optimization (identify right-sizing opportunities aligned to business demand), and Track and Communicate Compute Cost Savings (capture and report realized and projected savings).

## Two tiers of the same capability

As with Public Cloud, two Compute solutions exist at different depth: **Compute Cost Overview** delivers a foundational view of server spend and TCO through a focused report set (Infrastructure Summary, Server TCO), suited for early optimization conversations without operational complexity. **Compute Insights & Optimization** delivers operationally richer analytics — cost, capacity, utilization, unit cost, and application-level consumption — to answer advanced questions: which server platforms need expansion, retirement, or migration; how unit costs (per server-hour, per instance, per core, per GB) are trending; and where opportunities exist to move workload to public cloud.

## Why this matters in a client conversation

Compute is usually where an infrastructure optimization conversation starts, because server utilization data is often the easiest infrastructure signal to act on quickly — a client can identify overprovisioned servers well before the full TBM Model is mature. Positioning Compute Cost Overview as the fast starting point, with a clear upgrade path to Compute Insights & Optimization once the client wants application-level consumption detail, avoids over-scoping an early engagement.
