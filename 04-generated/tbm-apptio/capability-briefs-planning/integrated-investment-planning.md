# Integrated Investment Planning (IIP) — How IBM Apptio Planning Addresses Project & Portfolio Investment Value

## The capability

A recurring gap in traditional IT budgeting is that project spend and the ongoing operational
cost that project produces get tracked in different places — a project's build cost looks
reasonable in isolation, but nobody connects it to the Run cost it generates for years afterward.
Integrated Investment Planning exists specifically to close that gap.

**Important licensing note**: this capability is explicitly scoped to the **Apptio Planning
Standard** subscription, per the source documentation — worth confirming in any proposal that
references it.

## How IBM Apptio Planning covers it

IIP — also referred to as Project Planning within the product — "enables IT organizations to
understand the total cost and financial impact of their technology investments," regardless of
whether those investments are structured as projects, portfolio epics, or value streams (a direct
connection to the Value Streams capability documented for Costing Standard — Planning extends the
same investment-tracking discipline forward in time).

Four capabilities anchor it:

- **Integrated Expense Planning** — links any budget line item to a planned or existing
  investment, so all spend — CapEx or OpEx — is connected to a strategic initiative rather than
  sitting as an unattributed line item.
- **Labor Activity Planning** — allocates internal and external resources to investments by role
  or by name, with cross-charging support so "consuming organizations reflect the true cost of
  effort" rather than absorbing unattributed labor cost.
- **Build and Run Cost Planning** — the core mechanism that solves the original gap: planning the
  "full lifecycle costs of each investment, from initial delivery (Build) to ongoing operations
  (Run)," so an organization can anticipate the operational budget impact of today's initiatives
  before they're approved, not after.
- **Role-Based Access and Collaboration** — differentiated visibility and edit access for Product
  Managers, Project Managers, and PMO teams, so each stakeholder contributes within their scope
  without needing broad system access.

## Why this matters in a client conversation

Build vs. Run cost planning is the single most useful lens to lead with for any client that has
been burned by a project that looked cheap at approval time and then quietly inflated the
operating budget for years afterward. This capability directly targets that failure mode — it
forces the Run cost projection to exist at planning time, not as a surprise discovered a year into
production.

The explicit support for "projects, portfolio epics, or value streams" as investment structures is
also worth highlighting for clients with agile or product-centric delivery models — IIP isn't
built exclusively around traditional waterfall project structures, it accommodates the funding
models increasingly common in modern technology organizations.
