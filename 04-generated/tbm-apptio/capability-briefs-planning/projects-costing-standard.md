# Projects — How IBM Apptio Costing Standard Addresses This TBM Capability

## The capability

Projects gives visibility into IT project investments and how that spend aligns with solutions, services, and business priorities, consolidating project financial data and linking it to solution offerings and business value classifications.

Three personas anchor this capability: IT Finance, the Project Management Office (PMO, which monitors project budgets and portfolio balance), and Service or Solution Owners. Four out-of-the-box use cases are documented: Project Spend Alignment (map project investments to solution offerings and business value classifications), Project Spend Variance Analysis (compare actual spend against approved budget to catch overruns early), Run–Change or Run–Grow–Transform Classification for Projects (classify project spend to understand the balance between operational, growth, and transformational initiatives), and Project Cost Allocation (allocate project costs, including labor and vendor, toward the corresponding solution offerings or business units). A custom use case, Investment ROI for Projects, extends this to return-on-investment tracking.

## Why this matters in a client conversation

Projects connects directly to Labor Financials and Vendor Management — a typical project combines both cost types — and to Value Streams, since the Run–Grow–Transform classification is exactly the lens a CIO uses to show how technology investment splits between keeping the lights on and growing the business. This is often the capability that makes a Portfolio/PMO conversation land: instead of tracking project spend in a separate spreadsheet disconnected from the rest of the cost model, project investment shows up in the same Cost Pool → Resource Tower → Solution chain as every other capability in this knowledge base.
