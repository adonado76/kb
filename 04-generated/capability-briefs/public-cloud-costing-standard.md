# Public Cloud — How IBM Apptio Costing Standard Addresses This TBM Capability

## The capability

Public Cloud gives visibility into cloud consumption and spend across infrastructure services, applications, and business units, combining cost, usage, and allocation data to support governance, accountability, and optimization of cloud investment.

Three personas anchor this capability: IT Finance, FinOps/Cloud Center of Excellence (CCoE, which optimizes usage and manages governance), and Solution Owners. Four out-of-the-box use cases run in increasing order of maturity: Cloud Cost Allocation & Showback (foundational: allocate and track cloud cost by department, team, environment, application, product, or service), Cloud Usage by Application (intermediate: tag and associate cloud usage with applications to reduce sprawl), Pre-purchased Cloud Capacity Management (intermediate: manage pre-purchased capacity such as reserved instances), and Cloud Cost Optimization (advanced: track, compare, and control spend to identify inefficiencies).

## Two distinct offerings — a scoping decision that matters

A detail worth getting right in any proposal: Apptio offers two distinct Public Cloud capabilities, and choosing the right one depends on client maturity.

**Public Cloud Integrated** combines cloud provider billing with enterprise financial data from the general ledger, incorporating both provider costs (AWS, Azure, GCP invoices) and non-provider costs (labor, support, shared services) allocated to the Cloud Resource Tower — the option for organizations that need full cloud TCO embedded in the broader TBM model, with more implementation effort due to model and allocation integration.

**Public Cloud for IT Finance** is, by contrast, a lightweight, standalone solution focused exclusively on provider spend, without incorporating non-provider costs — optimized for speed (activatable in days, not weeks), designed for Finance-led teams that need quick visibility without full model integration complexity. Apptio's own comparison table is explicit: only Public Cloud Integrated supports full cloud TCO, advanced allocation, and TBM integration; Public Cloud for IT Finance wins on time-to-value ("very fast" vs. "moderate") and low implementation complexity.

## Why this matters in a client conversation

This is a natural early-adoption entry point for cloud-heavy organizations: a client can stand up Public Cloud for IT Finance in days to get immediate provider-spend visibility, and graduate to Public Cloud Integrated once the broader TBM Model (Cost Pools, Resource Towers) is mature enough to absorb the additional allocation complexity. Positioning the two as a maturity path — not a one-time choice — avoids over-selling integration effort to a client that just needs fast visibility today.
