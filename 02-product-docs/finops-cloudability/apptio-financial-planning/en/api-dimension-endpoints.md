# Dimension Endpoints

*APIs used to manage Cloudability and Plan-only dimensions associated with CFP plans.*

## Dimension Object

| Field | Type | Description |
| --- | --- | --- |
| adjustment_attribute_id | string (UUID) | Legacy field for backward compatibility with AttributeDto (used when denoting existing plan dimensions) |
| data_type | string | Data type of the dimension. Possible values: List, Integer, Decimal, Date, String |
| group | object | Grouping information for the dimension |
| id | string (UUID) | Unique identifier of the dimension |
| label | string | Display name of the dimension |
| metamodel_entity_id | string (UUID) | Legacy field for backward compatibility. For example, a plan’s costOwnerDimensionId corresponds to a dimension’s metamodel_entity_id |
| name | string | Internal name of the dimension |
| options | array | Possible values (if dimension type is List) |
| sub_group | object | Sub-group information for the dimension |
| type | string | Dimension type: Cloudability or Plan |

Example Dimension Object

```
{
  "adjustment_attribute_id": "01ef63c7-bbeb-47d8-9e29-b1ca51672560",
  "data_type": "List",
  "group": {
    "id": 2147483647,
    "key": "plan_only",
    "name": "Plan-Only"
  },
  "id": "edc820df-a294-402f-87c3-d1a40180bace",
  "label": "My Dimension",
  "metamodel_entity_id": "e5ae2a9b-e3ea-4fb7-baf3-60dbe84d9439",
  "name": "custom_my_dimension",
  "options": [],
  "sub_group": {
    "id": 2147483647,
    "key": "plan_only",
    "name": "Plan-Only"
  },
  "type": "Plan"
}

```

## Get Dimensions

Retrieves a list of available dimensions.

Endpoint

```
GET /v3/planning/dimensions?type={type}&userId={userId}
```

Request Example

```
curl 'https://api.cloudability.com/v3/planning/dimensions?type=Plan&userId=123456' \
  -H "Accept-Language: en-US" \
  -H "Apptio-Current-Environment: [apptio Frontdoor environment id]" \
  -H "Apptio-Opentoken: [apptio opentoken]"

```

Parameters

| Parameter | Type | Description |
| --- | --- | --- |
| type | enum | Type of dimensions: Plan or Cloudability |
| userId | integer | Cloudability user ID of the requestor |

## Create a Plan Dimension

Creates one or more plan-only dimensions. Each dimension is defined by an entry in the specs array.

Endpoint

```
POST /v3/planning/dimensions
```

Request Example

```
curl -X POST 'https://api.cloudability.com/v3/planning/dimensions' \
  -H "Content-Type: application/json" \
  -H "Accept-Language: en-US" \
  -H "Apptio-Current-Environment: [apptio Frontdoor environment id]" \
  -H "Apptio-Opentoken: [apptio opentoken]" \
  -d @- << EOF
{
  "specs": [
    { "data_type": "Text",   "label": "Project Code",     "type": "Plan" },
    { "data_type": "Numeric","label": "Headcount",        "type": "Plan" },
    { "data_type": "Data",   "label": "Activation Date",  "type": "Plan" },
    { "data_type": "List",   "label": "Cost Center",      "type": "Plan" }
  ]
}
EOF

```

Parameters

| Parameter | Type | Description |
| --- | --- | --- |
| specs | array<Spec> | Array of dimension definitions. Each object creates one Plan-only dimension. See Spec (object) below. |

Spec (object)

| Field | Type | Description |
| --- | --- | --- |
| data_type | enum | Data type of the dimension. Allowed values: Text, Numeric, Data, List. |
| label | string | Human-readable name of the dimension being created. |
| type | enum | Must be Plan. This endpoint only creates Plan-only dimensions. |

- This API does not create Cloudability dimensions; type must be Plan

- For data_type = List, you can populate allowed values after creation using Update a List Dimension’s Options ( PATCH /v3/planning/dimensions/{dimensionId} ) with an options array.
- label is the display name your users will see in the UI; an internal name is generated by the service

## Add Dimension to a Plan

Associates existing dimensions with a specific plan. You can add both Cloudability and existing Plan-only dimensions. This endpoint does not create new dimensions; it only adds existing ones to the plan.

Endpoint

```
POST /v3/planning/plans/{planId}/dimensions?viewId={viewId}
```

Request Example

```
curl -X POST 'https://api.cloudability.com/v3/planning/plans/{planId}/dimensions?viewId=0' \
  -H "Content-Type: application/json" \
  -H "Accept-Language: en-US" \
  -H "Apptio-Current-Environment: [apptio Frontdoor environment id]" \
  -H "Apptio-Opentoken: [apptio opentoken]" \
  -d @- << EOF
{
  "specs": [
    { "name": "account_name","type": "Cloudability" },
    { "name": "project","type": "Plan" },
    { "name": "project_start","type": "Plan" }
  ],
  "userId": 123456
}
EOF
```

Parameters

| Parameter | Type | Description |
| --- | --- | --- |
| planId | string (UUID) | Plan to which dimensions will be added |
| viewId | integer | Cloudability View ID of the plan |
| specs | array<Spec> | Array of existing dimension references to add (see Spec (object)) |
| userId | integer | Cloudability user ID of the requestor |

Spec (object) – for Adding Dimensions

| Field | Type | Description |
| --- | --- | --- |
| name | string | The name of the existing dimension to add. |
| type | enum | Cloudability or Plan. |

Notes:

- To create new Plan-only dimensions, use Create a Plan Dimension (with specs[].data_type, specs[].label, specs[].type="Plan").
- After adding a List-type Plan dimension, you can populate its values with Update a List Dimension’s Options .

## Export Plan Dimension Values

Exports dimension values for a specific plan dimension to a CSV.

Endpoint

```
POST /v3/planning/plans/{planId}/dimensions/{dimensionId}/options/export?viewId={viewId}
```

Request Example

```
curl -X POST 'https://api.cloudability.com/v3/planning/plans/{planId}/dimensions/{dimensionId}/options/export?viewId=0' \
  -H "Content-Type: application/json" \
  -H "Accept-Language: en-US" \
  -H "Apptio-Current-Environment: [apptio Frontdoor environment id]" \
  -H "Apptio-Opentoken: [apptio opentoken]" \
  -d @- << EOF
{
  "characterEncoding":"UTF_8",
  "columnDelimiter":"COMMA"
}
EOF
```

Parameters

| Parameter | Type | Description |
| --- | --- | --- |
| planId | string (UUID) | ID of the plan |
| viewId | integer | Cloudability View ID of the plan |
| dimensionId | string (UUID) | ID of the dimension |
| characterEncoding | enum | Encoding of the CSV (e.g., UTF_8 ) |
| columnDelimiter | enum | Delimiter of the CSV (e.g., COMMA ) |

## Update Plan's List Dimension Values

Updates the allowed values of a list-type dimension used within a Plan. This API only updates the instance of the dimension used within the plan. Use Update Global List Dimension Values to update the global definition.

Endpoint

```
PUT /v3/planning/plans/{planId}/dimensions/{dimensionId}?viewId={viewId}
```

Request Example

```
curl -X PUT 'https://api.cloudability.com/v3/planning/plans/{planId}/dimensions/{dimensionId}?viewId=0' \
  -H "Content-Type: application/json" \
  -H "Accept-Language: en-US" \
  -H "Apptio-Current-Environment: [apptio Frontdoor environment id]" \
  -H "Apptio-Opentoken: [apptio opentoken]" \
  -d @- << EOF
{
  "allowUsersToAddValues":false,
  "defaultValue":null,
  "isRequired":true,
  "options":["option 1", "option 2"]
}
EOF
```

Parameters

| Parameter | Type | Description |
| --- | --- | --- |
| planId | string (UUID) | ID of the plan |
| viewId | integer | Cloudability View ID of the plan |
| dimensionId | string (UUID) | ID of the dimension |
| allowUsersToAddValues | boolean | Whether users can add new values |
| defaultValue | string or null | Default value if applicable |
| isRequired | boolean | Whether the dimension is required |
| options | array | List of allowed values |

## Update Global List Dimension Values

Updates the allowed values of a Plan-only list-type dimension. This only updates the global definition of the dimension and does not affect any of the instances of the dimension currently used in plans.

Endpoint

```
PATCH /v3/planning/dimensions/{dimensionId}
```

Request Example

```
curl -X PATCH 'https://api.cloudability.com/v3/planning/dimensions/{dimensionId}' \
  -H "Content-Type: application/json" \
  -H "Accept-Language: en-US" \
  -H "Apptio-Current-Environment: [apptio Frontdoor environment id]" \
  -H "Apptio-Opentoken: [apptio opentoken]" \
  -d @- << EOF
{
  "options":["option 1","option 2"]
}
EOF

```

Parameters

| Parameter | Type | Description |
| --- | --- | --- |
| dimensionId | string (UUID) | ID of the dimension to update |
| options | array | New list of allowed values |

## Make a Cloudability Dimension Recommended For All Plans

Marks one or more Cloudability dimensions as “recommended” across all plans. This influences the suggested defaults when configuring plan dimensions, but does not force-add dimensions to existing plans

Endpoint

```
POST /v3/planning/dimensions
```

Request Example

```
curl -X POST 'https://api.cloudability.com/v3/planning/dimensions' \
  -H "Content-Type: application/json" \
  -H "Accept-Language: en-US" \
  -H "Apptio-Current-Environment: [apptio Frontdoor environment id]" \
  -H "Apptio-Opentoken: [apptio opentoken]" \
  -d @- << EOF
{
  "specs": [
    { "name": "business_unit", "type": "Cloudability" },
    { "name": "account_name", "type": "Cloudability" }
  ],
  "userId": 123456
}
EOF
```

Parameters

| Parameter | Type | Description |
| --- | --- | --- |
| specs | array<Spec> | Array of existing dimension references to add (see Spec (object)) |
| userId | integer | Cloudability user ID of the requestor |

Spec (object) – for Recommending Cloudability Dimensions

| Field | Type | Description |
| --- | --- | --- |
| name | string | The Cloudability dimension name (e.g., business_unit, account_name) |
| type | enum | Must be Cloudability for this endpoint |

- This endpoint does not create dimensions and does not add them to a specific plan. It only updates the recommended set shown during plan setup.
- To actually attach dimensions to a plan, use Add a Dimension to a Plan.
- To create new Plan-only dimensions, use Create a Plan Dimension with specs[].data_type, specs[].label, specs[].type="Plan" where data_type ∈ { Text, Numeric, Data, List }.

## Delete a Dimension

Deletes a dimension. For Cloudability dimensions, the dimension will be removed from the recommended set (but not from existing plans). For plan dimensions, the dimension is removed from all associated plans and values are lost.

Endpoint

```
DELETE /v3/planning/dimensions/{dimensionId}
```

Request Example

```
curl -X DELETE 'https://api.cloudability.com/v3/planning/dimensions/{dimensionId}' \
  -H "Accept-Language: en-US" \
  -H "Apptio-Current-Environment: [apptio Frontdoor environment id]" \
  -H "Apptio-Opentoken: [apptio opentoken]"

```

Parameters

| Parameter | Type | Description |
| --- | --- | --- |
| dimensionId | string (UUID) | ID of the dimension to delete |
