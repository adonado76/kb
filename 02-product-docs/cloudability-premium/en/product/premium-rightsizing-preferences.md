# Rightsizing Preferences

## About Preferences

This page allows you to setup global preferences for Cloudability rightsizing recommendations
preferences under “Basic” tab. As a part of Cloudability Premium, optimization actions generated by
the Turbonomic engine flow into Cloudability and these are shown under the Advanced tab in the
Optimize > Rightsizing page. A Cloudability Admin can now control if they want their users to see
the Cloudability generated rightsizing recommendations or hide them.

## Advanced Preferences

To setup visibility of Cloudability rightsizing recommendations, navigate to Settings >
Rightsizing Preferences > Advanced tab

![](../../../../03-media/cloudability-premium/images/advanced-rightsizing-preferences-hide.png)

The default option set is “Show All” which means the Optimize > Rightsizing page will show both
Cloudability generated rightsizing recommendations in “Basic” tab and the optimization actions
generated by the Turbonomic engine in “Advanced” tab. However, a Cloudability Admin can choose to
hide the Cloudability generated rightsizing recommendations which will grey out the “Basic” tab for
all users in the Optimize > Rightsizing page. Cloudability Admin can toggle this setting at any
time. While the rightsizing recommendations will continue to be generated by Cloudability, this flag
is only the visibility of the rightsizing recommendations in the Cloudability UI for all users.

## Advanced Configuration Settings

This section allows you to configure policies that influence/automate the optimization actions
generated by the Turbonomic engine and related configurations including
Groups, Schedules, Workflows
and Policies. These are presented as tiles. On click on any of these tiles,
it opens up the specific configuration page in a new tab in your browser.

- Groups: Groups assemble collections of resources for Turbonomic to
  monitor and manage. When you set the scope for your session connecting to the Turbonomic engine, you
  can select groups to focus on those specific resources. For example, if you have VMs devoted to a
  single customer, you can create a group that includes those VMs. You can then set the scope to that
  group when you create a policy or run a plan. Turbonomic engine supports dynamic and static
  groups.
- Schedules: Schedules specify a period of time during which certain events
  can occur. A calendar schedule is a setting that sets up a window of time when an automation policy
  takes effect. This policy can execute non-parking actions on public cloud or on-prem entities, or
  change settings that affect analysis and action generation.
- Workflows: Automation workflow specifies whether Turbonomic engine will
  execute an action, or whether Turbonomic engine will automate external workflows to effect the
  change in your environment. In this way, you can integrate supported orchestrators to execute of
  actions for specific scopes of entities in your environment. You can create an automation workflow
  that triggers a webhook whenever Turbonomic engine recommends moving a VM, or you can create an
  automation workflow that runs as a replacement for the action that Turbonomic engine would
  execute.
- Policies: Policies set business rules for resource allocation analysis,
  resource status display, and action execution. An automation policy is a set of rules that Turbonomic
  must satisfy when executing non-parking actions on public cloud or on-prem entities, or changing
  settings that affect analysis and action generation.

Note: You can only manage policies if you have relevant Turbonomic roles assigned to you in
Frontdoor

## Basic Preferences

To setup global preferences for Cloudability rightsizing recommendations, navigate to Settings > Rightsizing Preferences > Basic tab

Compute (VM) Preference

This preference tab allows you to set up global preferences for Cloudability rightsizing
recommendations.

![Compute VM preferences screenshot](../../../../03-media/cloudability-premium/images/basic_rightsizing_preferences_VM.png)

| Field Name | Description |
| --- | --- |
| Generations and Instances | Options for excluding non-current instance types or to recommend only within existing instance family (e.g. to ensure continued coverage with existing commitments / reservations) when generating rightsizing recommendations. |
| Processor Architecture | Choose to exclude specific processor types from rightsizing recommendations and allow cross -architecture recommendations. Ensure workload compatibility before switching the processor type. |
| Capacity Reduction | Options for including recommendations that would result in a reduction in resource capacity (e.g. reducing total available memory) if utilization metrics for the dimension are not provided. |
| Cost Saving Threshold | Optional setting for determining the minimum savings recommendations must be predicted to achieve in order to be included in those displayed.  Minimum 30-Day Savings Amount - A value of zero indicates the setting will be ignored. |
| Resource Lifespan | Optional setting for eliminating recommendations for the resources that have been inactive for a specified period of time. A value of zero indicates the setting will be ignored. |

Enter data in the appropriate fields and select the Save button. The message *Successfully saved preferences* appears.

Multiple ways to filter rightsizing recommendations

In Cloudability, there are many ways to filter rightsizing recommendations. First, you can filter the recommendations by using the global preferences available under Settings > Rightsizing Preferences page. Another way to filter using the options available on the rightsizing pages themselves under Optimize > Rightsizing. There are additional filtering options provided in the 'details' pane for the recommendations themselves.

Object Storage (S3) Preferences

This preference tab allows you to exclude specific classes for Object Storage for Rightsizing Recommendations.

To exclude a storage class:

1. select "Add Value" button.
2. Add a value (storage class) to the input field
3. Continue to add as may values to exclude as needed

S3 storage classes:

- Standard
- Intelligent Tiering
- Standard Infrequent Access
- Once Zone Infrequent Access
- Glacier Instant Retrieval
- Glacier Flexible Retrieval
- Glacier Deep Archive

## Frequently Asked Questions

1. Do I need to redo policy configurations in Cloudability UI in the Advanced tab
   view?

   Whatever policy configurations you may configured in your Turbonomic engine
   setup with your Cloudability Premium deployment, the same configurations will be available for you
   to manage via the Cloudability UI too now. You won't lose any of your existing configurations in
   Cloudability Premium Turbonomic engine.
2. Can I view and manage the same policies in Turbonomic UI too along with what is seen in the
   Advanced tab view?

   Yes, the same policy configurations can be managed via the Turbonomic UI
   (accessible through the Cloudability Premium tile in Frontdoor) too. We are only providing an ability to view
   and manage Turbonomic policy configurations via the Cloudability UI from Rightsizing Preference > Advanced page
   as an option.
3. Are the preferences set in Basic and Advanced tabs different?

   Yes,
   these rightsizing preferences are very different. While the one set in Basic
   tab is used by the Cloudability recommendation engine, the one in Advanced
   tab are used by the Turbonomic engine. They are neither interchangeable nor interoperable.
