# Action Execution

## Introduction

Once you have credentialed your cloud accounts completely, Turbonomic engine starts its analysis of
your environment. The holistic analysis by the Turbonomic engine identifies actions you can take to
optimize your cloud deployments, both in terms of performance and cost. These Turbonomic powered optimization
actions are displayed in the Rightsizing > Advanced page in Cloudability for you to investigate and act.

To get the best results from these Turbonomic powered actions, execute these actions promptly and consider
automating as many of them as possible.

## Action Execution and Automation

By default, Turbonomic engine does not execute actions automatically. If you examine the default
policies that ship with the product, you will notice that these policies do not enable automation on
any action. Turbonomic engine gives you full control over execution of these actions and all
automation decisions.

Your goal must be to evaluate all the actions in presented in the Rightsizing > Advanced page and then execute
actions that result in immediate improvements in performance/cost and resource utilization. Over time, you develop
and fine-tune your action-handling process to meet productivity goals and respond to changing business needs.
This process could lead to the following key decisions:

- Manully Execute Action: Continue to let Turbonomic engine post certain
  actions so you can execute them on a case-by-case basis.

  For example, certain actions might
  require the approval of specific individuals. In this case, you would want Turbonomic engine to post
  those actions for review and only execute the actions that receive an approval.

  These actions
  stop showing after you execute them, if you disable or automate them, or if the environment changes
  in the next market analysis such that the actions are no longer needed.
- Automate Actions through Automation Policies:  Automation policies allow
  certain actions to be executed automatically, such as those that assure performance and/or cost on
  mission-critical resources.

  Automation simplifies your task, while ensuring that workloads
  continue to have adequate resources to perform optimally. As such, it is important that you set the
  goal of automating as many actions as possible. This requires evaluating which actions are safe to
  automate, and on which entities.

## Manual Action Execution

If an action is executable, you can execute it from Action Details panel. Select the relevant
actions that you are interested in by choosing "View Details" on the action from the optimization
actions listing page, and then click Execute Action.

![](../../../../03-media/cloudability-premium/images/advanced-rightsizing-for-aws-ec2-listing.png)

Action details panel now allows you to execute this action.

![](../../../../03-media/cloudability-premium/images/advanced-rightsizing-for-aws-ec2-details-action.png)

When you choose to execute an action, you are prompted to confirm after which the action
execution starts. At this point in time, the Execute Action button is greyed
out with a tool tip text indicating the action execution is in progress. This action will continue
to remain in this state (i.e., Execute Action button greyed out in the
details page) until the next actions refresh cycle (60 mins).

Note: You can only execute an action if you have AutomatorFullAccess permission assigned to
you in Frontdoor

Action Execution - Points to Consider

- Some actions may not be executable. For example, the action acceptance setting in a policy may have been
  set to Recommend or the underlying technology for the entity does not support automation. In such cases, the
  Execute Action button may not be enabled for an action in the UI
- Some actions can only be executed after certain prerequisites are met. For example, to suspend Host A,
  VM\_01 in the host must first move to Host B. However, Host B only has capacity for one VM and is currently
  hosting VM\_02. In this case, Host A suspension is blocked by two prerequisite actions – VM\_02 moving to
  another host and VM\_01 moving to Host B. For an action with prerequisites, the Execute Action
  button is grayed out and the status indicates that it is blocked.

## Frequently Asked Questions

1. Can all Turbo actions be executed?

   No, not all optimization actions
   generated by the Turbonomic engine are executable. It depends on how policies are configured,
   underlying technology for the entity must support execution or all required per-requisites for an
   action have to be met.
2. How do I track the action execution status in UI?

   The status field
   in the action listing page as well as the action details panel reflect the current action execution
   status. Please note the status values in Cloudability do not immediately reflect the action
   execution status in Cloudability UI. However, users can navigate to the Turbonomic UI via Frontdoor,
   locate the resource in context from the supply chain view and look at the action status in the
   All Actions widget
3. I executed an action, what happens next?

   Once you execute an action
   in Cloudability UI, it triggers the action execution request in the Turbonomic engine. The
   Execute Action button in Cloudability UI will get greyed out to prevent this
   or any other users in Cloudability in executing the same action again. A tool tip text will indicate
   this action execution is in progress. This action will continue to be listed in Cloudability UI both
   in the listing page as well as the action details pages are visible to the users in Cloudability UI,
   though users cannot execute it. However, the status of the action execution in Turbonomic engine is
   not reflected in Cloudability UI. Users can track the execution request in Turbonomic UI through the
   All Actions widget. Cloudability refreshes actions from Turbonomic every 60
   minutes and in the next refresh cycle, this action may see updates in the Cloudability UI based on
   the Turbonomic assessment cycle
4. How do I troubleshoot the action execution flow? Where do I
   start?

   It may take a few seconds or minutes for an action execution to be complete in
   the Turbonomic engine. You can check the action execution status for the cloud resource in context
   in Turbonomic UI. Navigate to the resource page (from supply chain view, you can choose entity type
   or such) and search the resource name/id in context. Open the “Overview” tab. The widget “All
   Actions” lists all the actions executed (successful or failed) in the lifetime of the resource in
   context is displayed here.
5. I executed an action which was successful, I still see the action listed in the
   UI. How is this possible?

   Even if the action execution is successful, Turbonomic
   engine takes a few minutes to reconcile this status and update its internal supply chain. Once this
   status is internally updated across all Turbonomic engine services, the next action generation run
   in the Turbonomic engine will ensure the action is not listed as it was already action-ed (unless
   the policy was updated in the interim with different thresholds). Since Cloudability updates
   Turbonomic engine generated optimization actions every 1 hour, it takes 1-2 hours for the action to
   go away from the Rightsizing page even after an action was executed successfully
6. I executed an action and it failed, what happens next?

   Turbonomic
   engine takes a few minutes to reconcile this status and update its internal supply chain. Once this
   status is internally updated across all Turbonomic services, the next Turbonomic engine's analysis
   cycle will generate a set of actions based on the policies configured and this action will re-appear
   in the list of actions in the UI (unless the policy was updated in the interim with different
   thresholds).
