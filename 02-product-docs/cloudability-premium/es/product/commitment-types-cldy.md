# Tipos de compromiso en Cloudability

Cloudability admite los siguientes tipos de compromisos en los tres principales proveedores de servicios en la nube. Las siguientes tablas muestran los detalles.

AWS Tipos de compromiso admitidos

| Categoría | Tipo | Facturado | Comprado en | Ámbito |
| --- | --- | --- | --- | --- |
| Recurso | EC2 Instancia reservada | Uso o pérdida por horas | unidades | Tipo de instancia, región, sistema operativo |
| RDS Instancia reservada | Uso o pérdida por horas | unidades | Región, motor de base de datos |
| Abrir instancia reservada de búsqueda | Uso o pérdida por horas | unidades | Tipo de instancia, opción de pago, región |
| Redshift Nodos reservados | Uso o pérdida por horas | Nodos | Tipo de nodo, región |
| Nodos reservados de Elasticache | Uso o pérdida por horas | Nodos | Tipo de nodo, región, SO |
| Gastar | Plan de Ahorro Computacional | Uso o pérdida por horas | Coste por hora | Envíe su solicitud a EC2 a través de la familia de instancias, tamaño, AZ, Región, SO o tenencia, y también envíe su solicitud a Fargate o Lambda. |
| EC2 Plan de ahorro | Uso o pérdida por horas | Coste por hora | Familia de instancias y región |
| SageMaker Plan de ahorro | Uso o pérdida por horas | Coste por hora | Se aplica independientemente de la familia de instancias, tamaño, AZ, AWS Región o componentes. |

Azure Tipos de compromiso admitidos

| Categoría | Tipo | Facturado | Comprado en | Ámbito |
| --- | --- | --- | --- | --- |
| Recurso | Instancia reservada de máquina virtual | Uso o pérdida por horas | Cantidad | Tipo de instancia, región, sistema operativo |
| SQL Database e Instancia reservada gestionada | Uso o pérdida por horas | Cantidad | Nivel de perfeccionamiento, región, tipo de despliegue |
| Base de datos para MySQL Capacidad reservada | Uso o pérdida por horas | Instancia | Opción de implantación, instancia y región |
| Base de datos para PostgreSQL Capacidad reservada | Uso o pérdida por horas | Instancia | Opción de implantación, instancia y región |
| Azure Blob Storage Capacidad reservada | Uso o pérdida por horas | Tamaño ( 100Tb, 1Pb ) | Nivel de acceso, redundancia, tamaño y región |
| Cosmos DB Capacidad reservada | Uso o pérdida por horas | Unidades reservadas (RU/s) | Nivel |
| Pools SQL dedicados de Synapse Analytics | Uso o pérdida por horas | Unidades de almacén de datos (DWU) | Región |
| Caché para reservas Redis | Uso o pérdida por horas | Nodos (análogo a la capacidad total de caché | Tipo de nodo, región, SO |
| App Service Instancias reservadas | Uso o pérdida por horas | Cantidad | Instancia, nivel, región |
| Gastar | Plan de Ahorro Computacional | Uso o pérdida por horas | Coste por hora | Tipo de ámbito |
| Planes de precompra de Databricks | Créditos fungibles en toda la legislatura | 1 DBU = 1 $ | Familia de instancias y región |
| Planes de precompra de Synapse Analytics | Créditos fungibles en toda la legislatura | 1 SCU = 1 $ | Aplicable a 7 productos |
| Rendimiento provisionado de Microsoft Foundry | Uso o pérdida por horas | PTU, una unidad de uso de rendimiento normalizada | Región, tipo de implementación |

GCP Tipos de compromiso admitidos

| Categoría | Tipo | Facturado | Comprado en | Ámbito |
| --- | --- | --- | --- | --- |
| Recurso | Motor de cálculo CUD | Uso o pérdida por horas | vCPU\*, Memoria\*, disco SSD local, GPU | Tipo de máquina, región |
| Gastar | Motor informático Flex-CUD | Uso o pérdida por horas | Equivalente horario a la carta | Tipo de máquina |
| CUD de Cloud SQL | Uso o pérdida por horas | Equivalente horario a la carta | Familia de instancias y región |
