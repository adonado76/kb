# Communication Workshop

## Preparing your Communication data

1. If you haven’t already done so, load your communication data including:
   - Communication
2. Following Apptio standard
   practices, categorize each raw data set into the category of Communications
3. If appropriate, apply a date filter to your communication data.

## About the Communication Identifiers

The identifier for Communications Master Data set is Object Identifier by default. You must have
some way to identify each Communication device and use that as object identifier.

## About the Communication Keys

Some of the keys in the Communications Master Dataset are system generated and some are
user-defined. Any system generated key should not be altered. The predefined keys are:

- ITRT\_Communications Key
- Communications\_End User Devices Key
- Communications\_Servers Key
- Communications\_Services Key

## Map data to Communication Master Data

Map your Communications data to the Communications Master Data dataset. Most of the mapping is
self-explanatory at this point. In 12.7 you can now leverage the Column Map function in your raw
dataset to map to the Communications Master Data ([Map
Columns](https://community.apptio.com/docs/DOC-10561 "(Opens in a new tab or window)"))

## Communication Reports

Currently there are no reports that light up after adding data to Communications Master Data
dataset.
