# Time Tracking Getting Started

Time Tracking is enabled using the **CTF – Time Tracking** component. This component
allocates labor costs using project- and activity-level time reporting and integrates with the
Costing Standard cost model.

Time Tracking should be used when detailed time reporting is required for project labor
costing.

Note: Depending on the use case, customers typically use **either Time Tracking or Labor
Units**, as using both may require additional model customization.

**CTF- Time Tracking**

The CTF- Time Tracking component allocates labor costs based on project and activity-level
time reporting. It enables detailed analysis of project labor effort and cost.

- Allocate labor costs to projects and operational activities
- Support monthly, quarterly, and annual operational reviews
- Analyze project labor cost and effort

## Prerequisites

- CTF- Cost Source
- CTF- Labor

This component impacts both Cost and CapEx metrics in the model and requires population of
the **Time Tracking Master Data** table.

The Time Tracking component does not introduce any additional out-of-the-box reports.
Time-based labor data is used within existing labor and project reports to enhance labor
cost allocation and project-level analysis

## Master Data Set

- **Time Tracking Master Data**
- Contains project- and activity-level time data used to allocate labor costs to projects
  and operational activities.
