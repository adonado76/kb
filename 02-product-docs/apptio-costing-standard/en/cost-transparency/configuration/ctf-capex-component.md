# CTF - CapEx component: install and configure

Use the optional CTF - CapEx component to separate CapEx reporting from the OpEx that
reporting in previous versions of Costing Standard. If your organization doesn't load CapEx data
into your models, this component will prevent the display of empty CapEx fields in out-of-the-box
reports. This allows you to focus on OpEx data if you only use OpEx reporting.

Applies to: TBM Studio 12.8 and later, with Template v107 and later

## Introduction

The CapEx component provides insights into IT capital expenditures at the account, cost pool, and
cost center levels in the budget and latest forecast variance reports.

The CTF - CapEx component is not installed automatically with the standard Costing Standard
project.

## Use cases

Data from this component supports the following decisions:

- Manage capital expenditures to budget and forecast
- Identify change in capital expenditures month over month, quarter to quarter, and other periods of time
- Identify top capital expenditures
- Identify top budget and forecast variances

## Install the component

To install the component:

1. Open the **Costing Standard** project.
2. In the **Project** tab, click **Components**.
3. Click the **CTF - CapEx** component.
4. Click **Install**.

## Master data

For a description of the fields in the master data table, see the information on the CTF - CapEx
component page in the product. To display the page:

1. In the **Project** tab, click **Components**.
2. Click the **CTF - CapEx** component.

## Required and recommended fields

The required and recommended fields needed for the standard Costing Standard labor reports are
same as those listed in [Populate the Cost Source Master Data table](populatecostsource.html "The Cost Source Master Data table defines the data required for the cost and budget related reports. To populate the table, you must upload cost figures as well as budget figures. To populate the Cost Source Master Data table, upload two data sets: Cost Actuals and Cost Budget. Map the data sets to the Cost Source Master Data table.").
