# Metafields no cálculo de custos padrão

Os metacampos oferecem flexibilidade na forma como você vincula vários conjuntos de dados mestre (como origem de custos para ativos fixos, origem de custos para mão de obra e origem de custos para fornecedores).

Considere a relação entre a fonte de custos e os fornecedores. Um cliente pode usar a Sungard por dois motivos: recuperação de desastres e nuvem. Nesse caso, você deve ter duas linhas no conjunto de dados mestre de fornecedores da Sungard, pois as linhas serão mapeadas para duas torres de TI diferentes com porcentagens de alocação diferentes para cada linha. Você pode usar o metacampo para trazer dois motivos diferentes pelos quais a Sungard é um fornecedor para diferenciar as linhas ao usar a alocação de referência baseada em dados.

Como outro exemplo, você poderia mapear o centro de custo para os metacampos, o que significa que você terá uma lista exclusiva de fornecedores e centros de custo para mapear.
