# Análise de mainframe

Utilize este relatório para compreender os padrões de consumo de recursos, avaliar a utilização em todas as linhas de produtos, aplicativos e unidades de negócios, e identificar tendências de custos para o planejamento e a otimização da capacidade. Aplique filtros e visualizações disponíveis para se concentrar nos dados relevantes para sua análise e apoiar decisões eficazes de alocação de recursos.

Este relatório foi elaborado para ser utilizado pelos seguintes perfis:

- Planejadores de capacidade de mainframe
- Gerentes financeiros de TI
- Responsáveis pelas aplicações
- Administradores de sistema

## Elementos-chave

![](../../../../../03-media/apptio-costing-standard/resources/images/ct_images/nx-mf-analysis.png)

| Elemento | Descrição |
| --- | --- |
| Controles de filtro (1) | Cinco filtros permitem filtrar o relatório por nome da linha de produtos, nome do produto, nome do complexo de processadores, localização e ambiente. |
| Tabela de análise interativa do mainframe (2) | Esta tabela apresenta dados de utilização do mainframe com colunas como unidade de medida comercial, nome da linha de produtos, nome do produto, nome do aplicativo, MSU de " GCP ", MSU de " zIIP ", custo e volume de negócios. Você pode aprofundar a análise e filtrar os dados. |
| Painel de seleção de colunas (3) | Este painel permite mostrar ou ocultar colunas na tabela, tais como ID da linha de produtos, nome da linha de produtos, ID do produto, nome do produto, ID da aplicação, nome da aplicação, carga de trabalho, classe de serviço e ambiente. |
| Júri de seleção métrica (4) | Este painel permite selecionar quais métricas serão exibidas na tabela, tais como MSU d GCP, MSU d zIIP, Custo, Volume de Negócios, Unidade de Medida de Negócios e Custo por Unidade de Volume. |
| Seletor de período (5) | Mostra o período do relatório. Use este controle para selecionar diferentes períodos. |

## Respostas às perguntas

- Quais aplicativos consomem mais recursos do mainframe em toda a sua organização?
- Como varia o uso de recursos entre as diferentes linhas de produtos e unidades de negócios?
- Qual é a composição dos custos de uso do mainframe por aplicativo e produto?
- Qual é a proporção entre a capacidade de processamento utilizada por processadores de uso geral e a utilizada por processadores especializados ( zIIP )?
- Quais aplicativos poderiam se beneficiar da otimização da carga de trabalho ou da migração para processadores d zIIP?
- Como é possível alocar com precisão os custos do mainframe às diferentes unidades de negócios para fins de repasse de custos?
- Que tendências no consumo de recursos podem ajudá-lo a planejar as necessidades futuras de capacidade?

## Ações recomendadas

- Filtre os dados por linha de produtos, nome do produto ou aplicação para se concentrar em áreas específicas de interesse.
- Exporte os resultados da análise para compartilhá-los com as partes interessadas ou realizar análises adicionais em ferramentas externas.
- Compare o uso de recursos em diferentes períodos para identificar tendências e padrões sazonais.
- Identifique os aplicativos com alto consumo de MSU ( GCP ) que possam ser candidatos à migração para um processador d zIIP.
- Analisar solicitações com custos excepcionalmente elevados e investigar oportunidades de otimização.
- Estabeleça cronogramas de monitoramento regulares para acompanhar o consumo de recursos e as tendências de custos ao longo do tempo.
- Colaborar com os responsáveis pelas aplicações para discutir as conclusões e desenvolver estratégias de otimização.
