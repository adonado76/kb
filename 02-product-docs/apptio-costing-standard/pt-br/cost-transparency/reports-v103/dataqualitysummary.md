# Qualidade dos dados - Relatório resumido

Aplica-se a: Costing Standard 11.8.x em execução em TBM Studio v12 ou TBM Studio v11.

## Introdução

O relatório Resumo concentra-se nas tendências de custos não alocados ao longo do tempo.

![](../../../../../03-media/apptio-costing-standard/resources/images/ct_images/dataqualitysummary_1.png)

## Navegação

Qualidade dos dados > Resumo

## Funções

Este relatório foi elaborado para administradores de TBM.

## Objetivos

Use a parte superior do relatório para:

- Analisar as tendências de custos não alocados ao longo do tempo.
- Entenda a porcentagem de custos alocados nas camadas do modelo: Finanças para torres de TI, torres de TI para aplicativos e serviços e aplicativos e serviços para unidades de negócios.

Use a parte inferior do relatório para:

- Revisar as estratégias de alocação para objetos específicos no modelo (por exemplo, origem do custo para mão de obra, origem do custo para ativos fixos, etc.).
- Identificar custos não alocados por alocação.
- Veja a "descrição amigável" das estratégias de alocação.
- Verificar a pontualidade dos dados carregados.

## Perguntas respondidas

Use as informações apresentadas na parte superior do relatório para responder às seguintes perguntas:

- Todos os meus custos da origem de custos estão fluindo para o modelo de custos?
- Meus custos não alocados estão diminuindo ao longo do tempo à medida que melhoro os dados e refino as alocações?
- Onde devo concentrar minha atenção para corrigir as principais lacunas?

Use as informações apresentadas na parte inferior do relatório para responder às seguintes perguntas:

- Quais das minhas estratégias de alocação têm custos não alocados?
- Quais são minhas maiores lacunas para que eu possa priorizar e me concentrar nessas áreas?
- O que a estratégia de alocação está tentando realizar em termos comerciais?

## Próximas ações

- Se a variação for irrelevante, nenhuma preocupação ou ação será necessária.
- Clique em uma conta específica para ver os detalhes da transação.
- Clique em um centro de custo específico para ver os detalhes no nível da transação.
- Vá para o Studio para melhorar as alocações por meio de alterações no modelo e no conjunto de dados.
- Atualizar conjuntos de dados "obsoletos" com novos dados.
