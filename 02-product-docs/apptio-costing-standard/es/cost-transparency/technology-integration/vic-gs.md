# Información sobre proveedores: integración de Coups. Introducción

Coupa es un software de gestión de compras que contiene datos que pueden utilizarse para implementar Vendor Insights . Siga los pasos que se indican a continuación para realizar la implementación Vendor Insights utilizando los datos de Coupa del conector Coupa Datalink (Classic).

1. [Configurar Coupa para la gestión de proyectos ( Apptio )](config-coupa.html)
2. [Configurar el conector Coupa DataLink](configure_the_datalink_coupa_connector.html)
3. [Configurar los datos de Coupa para Apptio](coupa-configure-data.html)
4. [Instalar y configurar Vendor
   Insights utilizando datos de Coupa.](coupa-install.html)

**Se necesitan expertos en la materia.**

Se necesitan los siguientes expertos en la materia para llevar a cabo los pasos indicados anteriormente.

- Administrador de   
    
   Coupa Esta persona necesitará acceso para iniciar sesión en la configuración administrativa de Coupa y registrar Datalink (Classic) como una aplicación aprobada. Durante este proceso de configuración, el administrador de Coupa deberá recopilar varios ajustes de configuración y proporcionarlos a TBMA para completar la configuración del conector de Datalink (Classic).
- Administrador de gestión de negocios tecnológicos (TBMA)   
    
   La configuración Vendor Insights requiere un TBMA que pueda acceder TBM Studio para configurar el Vendor
  Insights componente. Además, el TBMA deberá poder iniciar sesión en la consola de administración de Datalink (Classic), que fue registrada por el administrador de Coupa como una aplicación autorizada en la consola de administración de Azure. En TBM Connect se proporcionan instrucciones para guiar a la TBMA a través del proceso de configuración.
