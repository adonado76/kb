# Metacampos en la norma de cálculo de costes

Los metacampos ofrecen flexibilidad a la hora de vincular varios conjuntos de datos maestros (como Fuente de costes a Activos fijos, Fuente de costes a Mano de obra y Fuente de costes a Proveedores).

Considere la relación entre la fuente de costes y los proveedores. Un cliente puede utilizar Sungard por dos motivos: recuperación ante desastres y nube. En este caso, debería tener dos filas en el conjunto de datos maestros de proveedores para Sungard, ya que las filas se asignarán a dos torres de TI diferentes con porcentajes de asignación diferentes para cada fila. Puede utilizar el metacampo para introducir dos razones diferentes Sungard es un proveedor para diferenciar las filas cuando se utiliza la asignación de referencia basada en datos.

Otro ejemplo: puede asignar centros de coste a los metacampos, lo que significa que tendrá una lista única de proveedores y centros de coste que asignar.
