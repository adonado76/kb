# Billing editions and Costing editions

*Billing is available in two editions. Each edition is associated with a different set of components and is designed for different levels of complexity and maturity.*

## Billing Essentials

Billing Essentials focuses on getting a clean, repeatable PxQ billing process in place with a smaller footprint.

Key points:

- Delivered through the Billing– Charge and Billing– Charge Reporting components.
- Exposed in the front end as a “Billing” report collection within the same project that runs Costing Essentials.
- Uses a focused set of models, datasets, and reports to: Define billable offerings. Capture consumption by consumer. Apply rates and generate charges. Publish invoice-style views and archives.
- Customizations are limited to creating new reports and revising existing reports.

Billing Essentials is intended for organizations that:

- Want a simpler, transparent billing or showback process.
- Prefer a smaller set of reports and configuration points.
- Are early in their ITFM / TBM journey or have limited complexity in their service catalog.

## Billing Standard

Billing Standard goes further with a wider set of domain components and more advanced capabilities.

Key points:

- Uses a family of components, such as: Foundation Units Applications Servers Storage End User Devices Cloud Projects Price Cost Variance Plan Variance Inter Company Transfer Pricing
- Exposed via a separate Billing endpoint in addition to the Costing project.
- Provides reporting and analysis across multiple dimensions: Detailed unit-rate and driver analysis. Cloud and infrastructure breakdowns. Application and project-centric billing views. Variance analysis (cost vs plan vs price). Cross-entity allocation and transfer pricing.

Billing Standard adds more scope and flexibility. It is intended for organizations that:

- Have broader or more complex billing requirements.
- Need domain-specific views (cloud, project, application, infrastructure, devices).
- Need to support scenarios like transfer pricing, advanced variance analysis, or detailed unit-rate optimization.
- Have a greater need for customization.
