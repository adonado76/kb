## Gerenciamento de emissões de tecnologia - 12 de dezembro de 2024

Hoje lançamos o Technology Emissions Management ("TEM"). O TEM foi projetado para ajudar as organizações a examinarem seu patrimônio de carbono de TI (na nuvem e no local) para atender às necessidades internas de sustentabilidade e dar suporte aos requisitos de relatórios.

Antes dessa versão, os usuários não conseguiam visualizar relatórios relevantes relacionados a medidas de sustentabilidade para reduzir os esforços de carbono.

Os clientes podem ver as métricas mensais e acumuladas para as emissões em kWh e CO2e.

![imagem](../../../../../03-media/apptio-tbm-studio/resources/images/studio_images/r-notes/tem1.jpg)

Os dados são ingeridos a partir de Cloudability e Costing (quando aplicável), para fornecer uma visão única das emissões de carbono. As emissões no local e na nuvem serão mostradas em painéis para fornecer uma visão completa dos esforços e do uso de ESG/Sustentabilidade, com suporte a várias personas.

![imagem](../../../../../03-media/apptio-tbm-studio/resources/images/studio_images/r-notes/tem2.jpg)

![painel](../../../../../03-media/apptio-tbm-studio/resources/images/studio_images/r-notes/tem3.jpg)

## Como esse recurso pode ajudá-lo

As organizações precisam de uma solução que ofereça uma visão única e uma metodologia defensável para medir de forma consistente as emissões de carbono da TI em todos os provedores de nuvem e no local. Elas também precisam monitorar as métricas de carbono e reduzir as emissões para atingir as metas de sustentabilidade.

O Technology Emissions Management ("TEM") permite que os clientes equilibrem o custo com as emissões de carbono para TI em toda a infraestrutura no local e em Public Cloud .

O TEM oferece ao cliente transparência em seus cálculos de emissões de carbono de TI, bem como flexibilidade para melhorar as áreas de dados que podem ser afetadas.

Para saber mais, entre em contato com o seu representante Apptio.

## Billing Essentials (BETA, v200 )

Esse recurso permite que os gerentes de unidades de negócios, os proprietários de serviços e os gerentes financeiros de TI comuniquem o consumo e o custo dos serviços faturáveis aos consumidores. Esse recurso automatiza o processo de cobrança de serviços de TI selecionados, permitindo que os usuários cobrem serviços com base em uma abordagem PxQ, definam taxas usando custos unitários e ajustem a recuperação prevista acima ou abaixo. Além disso, os usuários podem gerar e enviar faturas mensais aos consumidores e armazenar registros de faturamento e estorno para uma trilha de auditoria clara, simplificando o processo de faturamento e reduzindo o esforço manual

![imagem](../../../../../03-media/apptio-tbm-studio/resources/images/studio_images/r-notes/be-1.jpg)

## Labor Workbench for CT Configurations (BETA, v120 )

Para começar, os administradores podem instalar o componente "CTF- Labor Workbench", que inclui o relatório do Labor Workbench e tabelas editáveis. Os usuários finais podem então modificar o mapeamento de mão de obra para Torres de TI nos relatórios do site Apptio e usá-lo em conjunto com o Time Tracking. Um guia de configuração está disponível para fornecer instruções de configuração passo a passo.

## Aprimoramentos no painel de componentes

Os principais aprimoramentos a seguir foram feitos na seção Instalado do painel Componentes:

- **Indicador de carregamento** : Um "Loading. A mensagem "Please Wait" (Aguarde, por favor) agora é exibida ao expandir os pacotes de componentes, garantindo uma melhor experiência do usuário durante o tempo de carregamento.
- **Filtragem aprimorada** : Somente os componentes instalados mais recentes são exibidos, e as versões mais antigas não são mais mostradas na seção Disponível.
- **Componentes instalados ocultos** : os componentes instalados agora estão ocultos na seção Disponível, reduzindo a desordem e melhorando a usabilidade.
- **Atualizações de API** : Adicionamos uma nova API ou opção para dar suporte a essas alterações, aprimorando a experiência geral de gerenciamento de componentes.
- **Experiência simplificada para autores sem conteúdo** : Os autores sem conteúdo agora só veem as versões mais recentes de cada pacote na seção Disponível, facilitando a localização do que precisam.

## Tabela editável

- **Fluxo automático de dados para tabelas editáveis enriquecidas** - Essa atualização permite que os usuários desativem/ativem o fluxo de dados em tabelas de transformação enriquecidas, reduzindo a confusão e proporcionando mais controle sobre as atualizações de dados. Com essa alteração, os usuários podem modificar as tabelas de transformação de origem sem atualizar automaticamente a transformação da tabela editável enriquecida, garantindo que os números do modelo de custo não sejam afetados até que as alterações sejam explicitamente publicadas. Além disso, o Score ET Transform não será atualizado automaticamente, exigindo que os usuários publiquem explicitamente as alterações para ver as alterações da tabela subjacente. Para ativar esse recurso, navegue até **Configurações do projeto** e selecione a **opção Desativar fluxo de dados em transformações enriquecidas**.

  ![desativar o fluxo de dados](../../../../../03-media/apptio-tbm-studio/resources/images/studio_images/r-notes/ddf-1.jpg)

  Anteriormente, qualquer alteração na tabela de origem era automaticamente transferida para a tabela editável, sem a etapa de publicação. Isso afetou os números no modelo de custo para os clientes.

  Para saber mais, consulte [Desativar fluxo de datas](../data_studio/create-table-from-et.html).
- **Opção de programação recorrente para "Promover automaticamente para produção** " - Uma nova coluna "Promover automaticamente para produção" foi adicionada à tela Programação recorrente, para permitir que os administradores tenham controle sobre o gerenciamento de dados e facilitar o monitoramento e o gerenciamento mais eficientes de trabalhos recorrentes. A função "Auto Promote To Production" promove dados novos e atualizados do ambiente de preparação para o ambiente de produção com base nos requisitos do administrador. Para ativar essa coluna, navegue até Projeto > Ativar recursos e selecione a opção **Ativar promoção automática para produção para programações recorrentes de tabelas editáveis**.

  ![imagem](../../../../../03-media/apptio-tbm-studio/resources/images/studio_images/r-notes/ap-enable.jpg)

  Ao selecionar essa opção, o **campo** "Auto Promote To Production" (Promover automaticamente para produção) e a **coluna** "Auto Promote to Production" (Promover automaticamente para produção) são exibidos conforme mostrado.

  ![promoção automática](../../../../../03-media/apptio-tbm-studio/resources/images/studio_images/r-notes/ap-field.jpg)

  ![opção de promoção automática](../../../../../03-media/apptio-tbm-studio/resources/images/studio_images/r-notes/ap-column.jpg)

  Para saber mais, consulte [Recurring Publish](../admin/recurring-publish-et.html "Aplica-se a: 12.11.7 e posterior. A opção Recurring Publish (Publicação recorrente) permite que o TBMA agende programações de publicação recorrentes automáticas para tabelas editáveis que tenham tabelas de transformação. Quando esse recurso for ativado, a opção \"Publicar no período\" será convertida em uma programação padrão e será anexada no pipeline Tabela editável, em Transformar tabela. Todos os \"Publicar no período\" serão convertidos em agendamento padrão na página Recurring Schedule, e a transformação vinculada ao agendamento estará disponível na página Transform Table.").
- **Novas funções para exibir a última e a próxima hora de publicação no relatório -** Duas novas funções GetNextPublishTime e GetLastPublishTime foram adicionadas para exibir a última hora de publicação e a próxima hora de publicação da transformação na superfície do relatório para tabelas editáveis. Esse recurso permite que os usuários visualizem facilmente o histórico de publicações e garantam que suas alterações tenham sido propagadas com sucesso para a tabela de transformação.

  ![imagem](../../../../../03-media/apptio-tbm-studio/resources/images/studio_images/r-notes/et-fns.jpg)

  Para saber mais, consulte [Funções](../formulas-and-functions/functions/functions.html "Aplica-se a: TBM Studio 12.0 e posterior").

## Precision Calc - otimização consciente do tempo (EA Beta)

O recurso Precision Calc tem duas novas otimizações com esta versão.

**As otimizações com reconhecimento de tempo** aumentam a eficiência calculando apenas os períodos de tempo em que os dados foram atualizados ou afetados. Esse recurso beta de acesso antecipado requer a aprovação do gerente de produto para ativar a funcionalidade e é ativado ao definir biit.background.cacher.enable.timeaware.optimization como true.

**As otimizações** da coleção de relatórios aumentam a eficiência, eliminando a necessidade de realizar uma atualização de cálculo quando um relatório é adicionado ou removido de uma coleção de relatórios.

Antes dessa versão, todos os períodos de tempo abertos e quaisquer alterações nas coletas de relatórios eram recalculados.

## Pré-processador de dados para tabelas JSON (EA Beta)

Um novo serviço de pré-processador de dados foi desenvolvido para analisar tabelas JSON antes de serem carregadas no site TBM Studio. A análise de JSON é um processo de cálculo intensivo que é executado novamente toda vez que o sistema exige um cálculo desses dados. Ao analisar os dados fora do mecanismo de cálculo do BIIT, o sistema terá um desempenho melhor e os tempos de cálculo serão reduzidos. Esse recurso requer aprovação do produto para ser ativado. Uma implementação controlada será programada ao longo de 2024 para migrar os clientes para esse novo serviço.

A funcionalidade futura permitirá que a filtragem, o agrupamento e a categorização de dados sejam aplicados antes de serem carregados no site TBM Studio.

## Tabelas editáveis

- **Recurso Excluir todas as linhas:** Um novo recurso opcional, **Excluir todas as linhas**, foi introduzido no componente de relatório de tabelas editáveis. O uso desse botão excluirá todas as linhas das tabelas editáveis brutas, deixando a tabela em branco; e removerá todas as linhas adicionadas ou modificadas das tabelas editáveis enriquecidas, deixando a tabela no mesmo estado da tabela de origem. As alterações na tabela editável serão rastreadas e estarão disponíveis por meio do recurso "Mostrar alterações".

  Esse recurso é desativado por padrão. Para ativá-la, clique com o botão direito do mouse no componente de relatório da tabela editável, selecione **Propriedades** > **Avançado** e ative a opção **Permitir excluir todas as linhas**. O botão **Excluir todas as linhas** ficará visível entre os botões "Adicionar linha" e "Publicar" na superfície do relatório para usuários com a permissão necessária.

  Antes desse recurso, os administradores precisavam criar um botão com o Apptio Script que executava duas condições opostas; por exemplo, se A=true, deleterow()); se A=false, deleterow(). Isso só funcionava para tabelas editáveis brutas. Se as linhas existentes precisassem ser excluídas de uma tabela editável enriquecida, era necessário que um engenheiro excluísse as linhas no banco de dados backend.

  ![imagem](../../../../../03-media/apptio-tbm-studio/resources/images/studio_images/r-notes/del-all-rows-1.png)

  ![imagem](../../../../../03-media/apptio-tbm-studio/resources/images/studio_images/r-notes/del-all-rows-2.png)

  Para saber mais, consulte a seção Excluir todas as linhas em [Editar propriedades de tabelas](../reports/tables/edit-properties-tables.html "Aplica-se a: TBM Studio 12.0 e posterior").
- **Slicers em várias tabelas editáveis** : Um slicer pode ser usado para filtrar em mais de um componente de relatório de tabela editável, mesmo que eles sejam baseados em tabelas editáveis diferentes. O nome da coluna deve ser o mesmo em ambas as tabelas editáveis. Um exemplo na solução ApptioOne Demand Planning é quando as tabelas editáveis Volume Plan e Plan Status são exibidas no mesmo relatório. A seleção de uma ou mais opções no fatiador filtrará ambas as tabelas. Se um slicer for colocado em um objeto de relatório de grupo, somente as tabelas dentro do grupo serão afetadas.

  Antes desse recurso, o administrador precisava ter um fatiador acima de cada tabela, com o fatiador usando uma coluna da tabela editável de origem. Os usuários finais precisariam então selecionar o(s) valor(es) desejado(s) em ambos os slicers para filtrar as duas tabelas com os mesmos valores.

  ![imagem](../../../../../03-media/apptio-tbm-studio/resources/images/studio_images/r-notes/slicers-multiple.png)

A funcionalidade de **download** reflete o recurso "<clique com o botão direito do mouse> e abra no Excel. e se baseia na visualização do relatório filtrado.

  O botão Upload está desativado por padrão e deve ser ativado no modo de estúdio. Com o relatório verificado, clique com o botão direito do mouse no componente de relatório de tabela editável, selecione **Propriedades** > **Avançado** e ative a opção **Permitir upload**. O recurso Upload só está disponível para tabelas editáveis brutas e, no momento, não é compatível com tabelas editáveis enriquecidas.

Esse método não é visível nem facilmente acessível para usuários casuais.
