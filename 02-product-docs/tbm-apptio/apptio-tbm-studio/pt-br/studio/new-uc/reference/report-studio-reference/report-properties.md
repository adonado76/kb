# Propriedades do relatório

As propriedades dos relatórios controlam o comportamento, a aparência e a acessibilidade dos relatórios no TBM Studio. Essas configurações afetam a forma como os dados são exibidos, quem pode acessar os relatórios, como os períodos de tempo são tratados e como os relatórios aparecem quando impressos ou exportados. Esta referência fornece documentação completa sobre todas as opções de configuração no nível do relatório.

## Metadados do relatório

Os metadados do relatório definem as propriedades básicas de identificação e organização de um relatório. Essas configurações são definidas ao criar um relatório e podem ser alteradas posteriormente.

**Propriedades básicas**

|  |  |
| --- | --- |
| **Propriedade** | **Descrição** |
| Nome do relatório | O nome de exibição do relatório. Aparece no Explorador de Projetos, nas guias de relatórios e na navegação. Os nomes devem ser descritivos e seguir as convenções de nomenclatura da organização. |
| Descrição | Texto opcional que descreve o objetivo e o conteúdo do relatório. Exibido como uma dica de ferramenta quando os usuários passam o mouse sobre o nome do relatório. Use este campo para explicar quais dados o relatório contém e qual é a sua finalidade. |
| Coleta de relatórios | A estrutura organizacional do relatório. As coleções de relatórios agrupam relatórios relacionados e podem ter suas próprias configurações de permissão, que se aplicam aos relatórios salvos. |
| Visível por | Determina quais funções podem visualizar o relatório. As opções incluem “Todos” ou funções específicas. Esta configuração determina o acesso inicial quando o relatório é criado. |
| Criar como nível superior | Ao salvar um relatório criado por meio da navegação detalhada, essa opção coloca o relatório no topo da hierarquia de relatórios, em vez de aninhá-lo sob seu pai. |

**Configuração do tamanho da tela**

Ao criar um relatório, você escolhe o tamanho da tela que determina a largura padrão do layout do relatório:

|  |  |  |
| --- | --- | --- |
| **Opção** | **Largura** | **Ideal para** |
| Padrão | 1024 pixels | Monitores tradicionais, telas de tablet, relatórios com tabelas menos largas |
| Tela panorâmica | 1440 pixels | Monitores widescreen modernos, painéis com várias colunas, layouts complexos |

Observação: *O tamanho da tela afeta a área de edição padrão. Os componentes podem ultrapassar esses limites, mas podem exigir rolagem horizontal em telas menores.*
