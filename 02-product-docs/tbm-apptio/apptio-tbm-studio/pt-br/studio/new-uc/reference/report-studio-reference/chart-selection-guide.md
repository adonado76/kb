# Guia de seleção de gráficos

Use a seguinte matriz de decisão para selecionar o tipo de gráfico mais adequado aos seus dados e ao objetivo da análise:

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| **Propósito** | **Poucas categorias** | **Várias categorias** | **Baseado no tempo** | **Hierárquico** |
| Comparar valores | Coluna | Barras | Coluna, Linha | Mapa em árvore |
| Mostrar tendência | Linha | Linha | Tendência | N/A |
| Parte do todo | Torta, Donut | Barras Empilhadas | Coluna empilhada | Mapa em árvore |
| Explicar a variação | Queda de água | Queda de água | Queda de água | N/A |
| Comparação de várias métricas | Radar | Sobreposição | Sobreposição | N/A |
| Progresso dos KPIs | Medidor | N/A | Componente de KPI | N/A |

**Melhores práticas para a seleção de gráficos**

- Use gráficos de pizza apenas para mostrar partes de um todo e quando houver seis categorias ou menos.
- Prefira gráficos de barras em vez de gráficos de pizza quando for importante fazer comparações precisas.
- Use gráficos de linhas para dados de séries temporais em que a tendência seja mais importante do que os valores individuais.
- Use gráficos em cascata para explicar variações ou diferenças entre dois valores.
- Use mapas em árvore quando tiver dados hierárquicos e quiser mostrar tamanhos relativos e proporções.
- Use gráficos sobrepostos ao comparar duas métricas relacionadas que possam ter escalas diferentes.
- Use gráficos de radar com moderação e apenas ao comparar vários atributos entre alguns itens.
