# Componentes do relatório: Tabelas

## Visão geral

As tabelas são o principal componente de exibição de dados no Report Studio. Eles apresentam dados estruturados em linhas e colunas, oferecem recursos interativos como classificação e filtragem e disponibilizam amplas opções de configuração para formatação, cálculos e navegação. As tabelas podem exibir dados de objetos de modelo, tabelas de transformação ou tabelas editáveis.

Esta seção de referência fornece documentação abrangente sobre todas as opções de configuração das tabelas, organizadas por área funcional.

## Tipos de tabelas

O Report Studio oferece suporte a vários tipos de tabelas para atender a diferentes necessidades de apresentação de dados.

|  |  |  |
| --- | --- | --- |
| **Tipo de tabela** | **Descrição** | **Caso de uso** |
| Tabela de dados padrão | Tabela básica com linhas, colunas e valores. Criado ao arrastar campos do Explorador de Projetos para as áreas de Configuração de Componentes. | O tipo de tabela mais comum. Utilize para exibir dados de custos, estoques, alocações e relatórios gerais. |
| Tabela agrupada | Consolida os dados por valores em uma ou mais colunas. Agrupa automaticamente as entradas duplicadas e exibe subtotais. | Resumo dos custos por unidade de negócios, serviço ou categoria. Comparando valores agregados. |
| Tabela em árvore de subtotais | Tabela hierárquica com linhas expansíveis. Os usuários podem navegar por vários níveis. | Discriminações hierárquicas de custos. Detalhamento por unidade de negócios, por aplicativo e por serviço. |
| Tabela dinâmica | Exibe dados dinâmicos utilizando as propriedades Pivot Row Col, Pivot Col Col e Pivot Val Col. | Análise de tabulação cruzada. Comparações temporais entre categorias. |
| Tabela editável | Permite que os usuários insiram e modifiquem dados diretamente no relatório. Suporta validação e transporte de saldos. | Lançamentos orçamentários, ajustes nas previsões, alocações manuais de custos e coleta de dados. |
