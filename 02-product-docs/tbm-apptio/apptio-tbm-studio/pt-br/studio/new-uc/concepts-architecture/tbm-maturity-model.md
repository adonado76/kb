# Modelo de Maturidade TBM

O TBM é uma jornada, não um destino. As organizações geralmente passam por fases de crescente sofisticação. Entender em que ponto você se encontra nessa jornada ajuda a definir expectativas realistas e a planejar seus próximos passos. O modelo de maturidade a seguir fornece um roteiro.

## Nível 1: Visibilidade dos custos

**Características**

Nesta fase, a organização tem uma noção básica dos custos de TI, mas possui capacidade limitada para detalhá-los por serviço ou usuário. Os dados de custos estão disponíveis nos sistemas financeiros, mas não estão estruturados para análise de TBM.

**Recursos típicos**

- O total dos gastos com TI é conhecido em termos gerais
- Os custos podem ser classificados em grandes categorias (hardware, software, mão de obra)
- Repartição limitada ou inexistente dos custos compartilhados
- A geração de relatórios é pontual, geralmente em planilhas

**Recursos do TBM Studio que oferecem suporte a este nível**

- Data Studio para importar e organizar dados financeiros de sistemas de contabilidade geral
- Tabelas de conversão básicas para classificação de custos
- Relatórios simples que mostram a composição dos custos por categoria

**Sinais de que você está pronto para avançar**

- Você tem dados consistentes e confiáveis sendo importados para o TBM Studio
- As partes interessadas estão pedindo mais detalhes (“O que está por trás desse número de software?”)
- Você identificou os principais fatores que poderiam apoiar a alocação

## Nível 2: Transparência de custos

**Características**

A organização implementou a alocação de custos e consegue rastrear os custos de TI, por meio das camadas da taxonomia, até os serviços e os usuários. O sistema de relatórios de reembolso está em vigor, e as partes interessadas sabem exatamente o que estão pagando pela TI.

**Recursos típicos**

- Alocação total de custos, desde os centros de custo, passando pelas estruturas organizacionais, até os serviços e unidades de negócios
- O cálculo de custos por nível de serviço está disponível
- Os relatórios de showback são distribuídos regularmente
- Já está em vigor o acompanhamento do orçamento em relação aos resultados reais

**Recursos do TBM Studio que oferecem suporte a este nível**

- O mecanismo completo de alocação do Model Studio com fluxos em várias camadas
- Várias métricas do modelo (custo, orçamento, previsão)
- Painéis do Report Studio com segurança por linha para relatórios de unidades de negócios
- Relatórios de modelo (visualizações de Sankey) para visualização do fluxo de custos

**Sinais de que você está pronto para avançar**

- Seu modelo de custos foi validado e as partes interessadas confiam nos números
- As unidades de negócios estão se perguntando: “Como podemos reduzir nossos custos?”
- Você tem dados históricos suficientes para a análise de tendências

## Nível 3: Otimização de custos

**Características**

A organização utiliza ativamente os dados do TBM para identificar e buscar reduções de custos. A análise de tendências revela padrões, a comparação com referências destaca oportunidades e as iniciativas de otimização são acompanhadas e avaliadas.

**Recursos típicos**

- Análise de tendências período a período
- Avaliação comparativa interna e externa
- Programas de otimização ativos baseados em dados de TBM
- A cobrança retroativa pode ser aplicada para criar incentivos financeiros
- Modelagem de cenários hipotéticos para as mudanças propostas

**Recursos do TBM Studio que oferecem suporte a este nível**

- Métricas calculadas para análise de variância e cálculo de tendências
- Tabelas publicadas para exportação de dados para sistemas financeiros (suporte a estornos)
- Tabelas editáveis para planejamento de cenários e enriquecimento de dados
- Recursos de detalhamento para análise da causa raiz

**Sinais de que você está pronto para avançar**

- A otimização de custos está gerando economias mensuráveis
- A liderança questiona o valor comercial dos investimentos em TI, e não apenas o custo
- Você dispõe da infraestrutura de dados necessária para vincular os custos de TI aos resultados comerciais

## Nível 4: Gestão de valor

**Características**

A organização relaciona os custos de TI ao valor comercial. As decisões de investimento são baseadas em dados do TBM, e a TI é vista como um parceiro estratégico de negócios, e não como um centro de custos. Os dados do TBM orientam a gestão de portfólio, a definição de prioridades de investimento e o planejamento estratégico.

**Recursos típicos**

- Gestão do portfólio de investimentos em TI
- Métricas de valor comercial associadas a dados de custos de TI
- Apoio estratégico à tomada de decisões sobre investimentos em tecnologia
- Cultura de otimização contínua incorporada à organização
- Previsão avançada e análise preditiva

**Recursos do TBM Studio que oferecem suporte a este nível**

- Conjunto completo de métricas de modelo, incluindo métricas personalizadas baseadas em valor
- Relatórios avançados com métricas calculadas que combinam dados de custos e de negócios
- Integração via API para conectar dados do TBM a sistemas de planejamento empresarial
- ApptioScript para cálculos personalizados e modelagem de valores

**Sinais de maturidade neste nível**

- As decisões de investimento em TI são baseadas em dados e estão ligadas aos resultados comerciais
- As conversas com os CIOs concentram-se na geração de valor, em vez de na redução de custos
- Os dados do TBM são integrados aos processos de planejamento e estratégia da empresa

## Resumo do Modelo de Maturidade

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| **Nível** | | **Foco** | **Pergunta-chave** | **Principais áreas do TBM Studio** |
| **1. Visibilidade** | | Saiba quanto você gasta | Quanto custa a TI? | Data Studio, relatórios básicos |
| **2. Transparência** | | Entenda como os custos são distribuídos | Em que estamos gastando nosso dinheiro? | Model Studio, visualizações de Sankey |
| **3. Otimização** | | Reduzir e otimizar custos | Onde podemos economizar? | Métricas calculadas, estorno, detalhamento |
| **4. Valor** | | Alinhar a TI com o valor comercial | Que valor agrega a TI? | Integrações completas com a plataforma e API |

Dica: A maioria das organizações opera nos níveis 2 ou 3. Não há necessidade de se apressar para chegar ao Nível 4. Cada etapa se baseia na anterior, e avançar muito rapidamente pode comprometer a confiabilidade dos seus dados de custos. Concentre-se em dominar o seu nível atual antes de avançar.
