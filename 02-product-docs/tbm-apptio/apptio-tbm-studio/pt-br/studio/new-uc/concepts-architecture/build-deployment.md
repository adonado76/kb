# Ciclo de vida de compilação e implantação

O TBM Studio utiliza um ciclo de vida controlado de compilação e implantação para garantir a integridade dos dados, facilitar a colaboração e proteger os ambientes de produção contra alterações não testadas. Compreender esse ciclo de vida é essencial para os administradores que gerenciam projetos e para qualquer usuário que faça alterações na configuração.

Esta seção explica como as alterações circulam pelos três ambientes do TBM Studio, como o fluxo de trabalho de check-out e check-in protege a integridade dos dados, como as compilações calculam seu modelo e como as alterações são implementadas na produção.
