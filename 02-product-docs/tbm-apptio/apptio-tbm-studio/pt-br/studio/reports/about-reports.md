# Sobre os relatórios

**Aplica-se a** : TBM Studio 12.0 e posterior

Para exibir dados aos usuários, crie relatórios usando a seção **Relatórios** no **Project Explorer**. Quando você edita um relatório, o painel **Configuração de componentes** e o espaço de trabalho do relatório são exibidos conforme mostrado na imagem abaixo. Os relatórios podem incluir uma variedade de componentes, incluindo, entre outros, tabelas, gráficos, botões, anotações, caixas de grupo e segmentações. Usando slicers, os usuários podem filtrar de forma rápida e fácil os dados exibidos em tabelas e gráficos. Para obter informações sobre como visualizar e imprimir relatórios, consulte [Trabalhar com relatórios](work-with-reports.html "Aplica-se a: TBM Studio 12.0 e posterior").

![imagem](../../../../../03-media/apptio-tbm-studio/resources/images/studio_images/studioimages/reports/studio_about_reports_panels_800x479.png)

## Adicionar interatividade

Os relatórios baseados em objetos nos modelos podem incluir links que levam a relatórios mais detalhados. Os links fornecem um nível básico de interatividade para os usuários. Você pode proporcionar mais interatividade adicionando slicers, seletores de campo e pivôs rápidos, conforme mostrado na imagem a seguir. Os fatiadores filtram a tabela (ou gráfico) pelos valores selecionados. Selecionar Column Pickers adiciona colunas e selecionar Quick Pivot agrupa os dados por uma coluna selecionada.

![imagem](../../../../../03-media/apptio-tbm-studio/resources/images/studio_images/studioimages/reports/studio_reports_interactivity.png)

## Tabelas e gráficos personalizados

Além de fatiadores, seletores de campo e pivôs rápidos, os usuários finais atribuídos a uma função de analista podem adicionar tabelas e gráficos personalizados aos relatórios.

## Etapas gerais para a criação de relatórios

As etapas gerais para a criação de relatórios estão listadas abaixo:

- Criar um relatório em branco.
- Adicione tabelas e gráficos.
- Organize as tabelas e os gráficos para melhor apresentar os dados e facilitar o acesso.
- Adicione slicers para que os usuários possam filtrar o relatório.
- Adicione navegação entre os relatórios, se necessário.
