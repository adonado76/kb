# Componentes del informe: Tablas

## Visión general

Las tablas son el principal componente de visualización de datos en Report Studio. Presentan datos estructurados en filas y columnas, admiten funciones interactivas como la ordenación y el filtrado, y ofrecen amplias opciones de configuración para el formato, los cálculos y la navegación. Las tablas pueden mostrar datos de objetos de modelo, tablas de transformación o tablas editables.

Esta sección de referencia ofrece documentación exhaustiva sobre todas las opciones de configuración de las tablas, organizadas por área funcional.

## Tipos de tablas

Report Studio admite varios tipos de tablas para satisfacer las distintas necesidades de presentación de datos.

|  |  |  |
| --- | --- | --- |
| **Tipo de tabla** | **Descripción** | **Caso de uso** |
| Tabla de datos estándar | Tabla básica con filas, columnas y valores. Se crea arrastrando campos desde el Explorador de proyectos a las áreas de configuración de componentes. | El tipo de tabla más habitual. Se utiliza para mostrar datos de costes, inventarios, asignaciones e informes generales. |
| Tabla agrupada | Consolida los datos por valores en una o varias columnas. Agrupa automáticamente las entradas duplicadas y muestra los subtotales. | Resumen de los costes por unidad de negocio, servicio o categoría. Comparación de valores agregados. |
| Tabla de subtotales | Tabla jerárquica con filas desplegables. Los usuarios pueden profundizar en varios niveles. | Desglose jerárquico de costes. Desglose de la unidad de negocio a la aplicación y al servicio. |
| Tabla dinámica | Muestra datos desglosados mediante las propiedades «Pivot Row Col», «Pivot Col Col» y «Pivot Val Col». | Análisis de tabulación cruzada. Comparaciones temporales entre categorías. |
| Tabla editable | Permite a los usuarios introducir y modificar datos directamente en el informe. Admite la validación y el traspaso. | Entradas presupuestarias, ajustes de previsiones, asignaciones manuales de costes y recopilación de datos. |
