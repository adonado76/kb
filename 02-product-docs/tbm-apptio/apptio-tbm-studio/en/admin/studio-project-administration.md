# About Studio project administration

Applies to: TTBM Studio 12.0 and later.

Administering projects includes creating projects, roles, and users, and tracking errors. To
administer projects in Apptio, you must have the default role of Power User or Admin,
or a custom role with appropriate permissions. To access Administration features, click the
Project tab. Administration concepts include:

- Projects - Projects group together data sets, metrics, models, and reports. When you
  create a new project, you can choose from these major types: Costing Standard, Custom. You can
  create multiple projects. For more information on creating projects, see [Create and delete
  projects](create-deleting-projects.htm "(Opens in a new tab or window)").
- Users and roles - You control access to Apptio by defining users and assigning them to
  roles. The Users and Roles features are available from the Project tab. For
  information on creating users and roles, see [Control access to the Apptio applications](control-access-apptio.htm "(Opens in a new tab or window)").
- Error messages - You can review error messages by clicking Errors on the
  Project. The Errors window displays all error messages. The error messages are useful
  when troubleshooting. For more information, see [Review error messages](review-error-messages.htm "(Opens in a new tab or window)").
- Domain settings - You can control general settings for a domain by
  clicking **Domain Settings** on the Project tab. The dialog is shown in
  the following image:

  ![](../../resources/images/studio_images/studioimages/studio/stu595.png)
- Domain security - You can control general security settings for a domain
  by clicking the Project tab and clicking Domain
  Security. The dialog is shown in the following image:

  ![](../../resources/images/studio_images/studioimages/studio/stu596.png)

  Note: A value of -1 in the Min password age field and the Max
  password age field means "unlimited." Leave the Min password age field set to -1.
