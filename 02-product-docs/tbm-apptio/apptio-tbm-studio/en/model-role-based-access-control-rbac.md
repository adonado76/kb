# Role-Based Access Control (RBAC)

The authorization layer in TBM Studio uses a role-based access control (RBAC) model. Rather than assigning individual permissions to each user, permissions are grouped into roles, and users are assigned to one or more roles. This simplifies administration and ensures consistency—when you need to change what a group of users can do, you modify the role rather than updating each user individually.
