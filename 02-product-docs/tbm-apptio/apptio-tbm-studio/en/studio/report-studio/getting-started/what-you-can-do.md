# How the New Report Studio Works

The New Report Studio follows a simple, structured workflow that separates report
creation from report consumption. This ensures reports are easy to build, maintain, and use at
scale.

This page explains the end-to-end flow and how different users interact with Report Studio.

**High-Level Workflow**

Report Studio works in four main stages:

1. Create
2. Configure
3. Organize
4. View & Interact

Each stage builds on the previous one.

**Create Reports**

Admins start by creating a report.

A report defines:

- The data to be displayed
- The components used (tables, KPIs, slicers etc.)
- The visualizations that represent the data

**Configure Components and Visualizations**

Once a report is created, admins configure how data appears and behaves.

This includes:

- Selecting data sources, metrics, and dimensions
- Configuring component properties
- Setting up visualizations (charts, tables, layouts)
- Defining interactions between components (Ex: slicers affecting charts)

Common configuration options that apply across components and visualizations are documented
separately to avoid duplication.

**Organize Reports into Collections**

After reports are created, admins can group related reports into report collections.

Report collections help:

- Organize reports by theme, audience, or use case
- Improve discoverability for end users
- Set permissions on a collection to control which users can see the reports it contains

**View and Interact with Reports**

End users access reports through the Report Viewer. In the Report Viewer, users can:

- Navigate between reports and report collections
- Interact with filters and pickers
- View data in multiple currencies
- Export reports to PDF

**Roles in the workflow**

Different roles participate at different stages:

- Admins – Primarily involved in creating, configuring, and organizing reports.
- End Users – Primarily involved in viewing, interacting with, and exporting reports.

For a detailed breakdown, see [User Roles &
