# Summary

Report customization transforms individual reports into a cohesive, professional reporting
environment. The four techniques in this section work together:

|  |  |  |
| --- | --- | --- |
| **Technique** | **Purpose** | **Who Can Configure** |
| Custom Perspectives | Organize fields for easier report building | Power User, Admin |
| Report Collections | Group related reports for navigation | Admin (create/delete), Users (add reports) |
| Master Reports | Provide consistent layout and branding | Power User, Admin |
| Report Permissions | Control who sees what | Admin (report-level), Power User (component) |

**Recommended workflow:**

1. Design your custom perspectives with the fields analysts need most
2. Create master reports for consistent branding
3. Build reports using perspectives and masters
4. Organize reports into logical collections
5. Configure permissions at collection, report, and component levels

This foundation enables you to maintain a scalable, secure reporting environment that
serves diverse audiences while remaining manageable.
