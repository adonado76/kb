# Report Permissions

## Overview

Report permissions manage control of who can view, edit, and administer reports within TBM Studio. The permission system operates at multiple levels—from domain-wide role assignments down to individual component visibility—allowing granular control over report access while maintaining security and governance requirements.

This section covers the complete permission architecture for reports, including the relationship between report permissions and row-level security (RLS), best practices for permission design, and troubleshooting common access issues.
