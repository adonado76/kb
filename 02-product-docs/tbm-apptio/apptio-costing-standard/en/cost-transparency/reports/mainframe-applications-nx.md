# Mainframe Applications

Use this report to understand application costs, evaluate how resources align with
business priorities, and track cost trends across product portfolios and application lifecycles.
Apply filters and available views to focus on the data relevant to your analysis and support
informed investment and budgeting decisions.

This report is designed for use by the following personas:

- Application Portfolio Managers
- Business Unit Leaders
- IT Financial Analysts
- Application Owners

## Key Elements

![](../../../../../03-media/apptio-costing-standard/resources/images/ct_images/nx-mf-apps.png)

| Element | Description |
| --- | --- |
| Filter controls (1) | Two filters let you narrow the report by application family and application investment objective. |
| Spend by product portfolio chart (2) | This horizontal bar chart shows cost by product portfolio, such as Credit Cards, Customer Engagement, Wealth Management, Loans, and Finance and Actuarial. |
| Spend by application investment objective chart (3) | This vertical bar chart shows cost by application investment objective, such as Tolerate, Migrate, and Invest. |
| Spend by application lifecycle chart (4) | This horizontal bar chart shows cost by application lifecycle stage, with In Service being the primary category. |
| Column selection panel (5) | This panel lets you show or hide columns in the table, such as Application Business Owner, Application IT Owner, Application Family, Application Investment Objective, Application Lifecycle, Application Migration Strategy, and Application ID. |
| Mainframe application details table (6) | This table shows mainframe application data with columns such as application name, business unit of measure, volumes, unit cost, application family, application business owner, application IT owner, and cost. Application names link to more detailed information. |

## Questions Answered

- Which product portfolios consume the most budget for mainframe applications?
- How much are you spending on applications marked for migration versus those you plan to invest in?
- What is the unit cost for each application, and which applications have the highest cost per transaction?
- How are application costs distributed across different business owners and IT owners?
- Which applications are in service versus other lifecycle stages, and what are their associated costs?
- How can you identify applications that might benefit from modernization or retirement?
- What is the relationship between application volumes and costs across your portfolio?

## Recommended Actions

- Filter by application family or investment objective to focus on specific areas of your portfolio.
- Sort applications by cost or unit cost to identify the most expensive applications.
- Review applications with high unit costs to determine if optimization is possible.
- Compare spending across different product portfolios to ensure balanced investment.
- Identify applications marked for migration and plan modernization initiatives.
- Export the data to share with stakeholders or perform additional analysis.
- Schedule regular reviews with application owners to discuss cost trends and optimization strategies.
