# Conceitos-chave e terminologia

Compreender os conceitos básicos de IBMApptio Cálculo de custos ajudará você a navegar pelo aplicativo. Abaixo estão as definições dos principais termos frequentemente usados em Cálculo de Custos.

**AVF ( Apptio Value Framework)** - O AVF apresenta os principais casos de uso de Gestão Financeira de TI (ITFM) suportados pelo IBMApptio. Ele organiza cada caso de uso em torno de uma descrição clara do problema, os recursos do Costing Standard que o resolvem e as principais pessoas envolvidas. A AVF ajuda as equipes a entender onde se concentrar, como o valor é entregue e como as diferentes partes interessadas se beneficiam do IBMApptio Padrão de Custeio.

**ATUM (Modelo Apptio Unificado TBM)** - ATUM é IBMApptio o modelo de custos padronizado que alinha os dados financeiros, operacionais e de consumo à taxonomia TBM. Ele fornece uma estrutura consistente para alocar custos de tecnologia e produzir insights defensáveis e comparáveis entre as organizações.

**TBM (Gestão de Negócios Tecnológicos)**

O TBM é uma estrutura industrial que padroniza a forma como as organizações gerenciam, medem e comunicam o custo, a qualidade e o valor da tecnologia. Os princípios do TBM sustentam a estrutura da Norma Apptio de Custeio e sua metodologia de alocação de custos.

**Ambiente (Cliente/Servidor)** - Refere-se à infraestrutura que hospeda seu projeto Costing Standard. As atualizações do cliente e do servidor garantem compatibilidade, desempenho e acesso a novos recursos.

**Projeto** - Um projeto é o ambiente de trabalho onde os dados, componentes, modelo, alocações e relatórios do seu Padrão de Custeio são configurados e mantidos. Quando uma instância do Padrão de Custeio é provisionada, uma das primeiras ações a serem realizadas é criar o projeto.

**Modelo** - Um modelo é uma porta de entrada para conteúdo pré-criado a partir de IBM Apptio. Ao selecionar e definir o modelo no seu projeto, você terá acesso a conteúdos prontos para uso, como componentes. Os modelos simplificam a implementação, promovem a consistência e mantêm o alinhamento com futuras atualizações.

**Componente** - Um componente é um pacote modular instalável, como Mão de obra, Fornecedores, Servidores, Armazenamento, Aplicativos, Serviços, Mainframe etc., que contém conjuntos de dados mestres (incluindo lógica de fórmulas integrada), tabelas de referência (às vezes tabelas editáveis), lógica de alocação e relatórios pré-construídos.

**Conjunto de dados mestres (MDS)** — Os conjuntos de dados mestres são conjuntos de dados selecionados que servem como fonte oficial para documentos importantes, como fontes de custos, torres de recursos, mão de obra, fornecedores, ativos, aplicativos e serviços. As tabelas MDS normalizam as entradas de vários sistemas para garantir uma alocação e relatórios consistentes.

**Modelo** - Um modelo define como os custos de tecnologia são estruturados, transformados e alocados dentro de um projeto. Ele integra várias fontes de dados, expostas por meio de conjuntos de dados mestres, ao mesmo tempo em que aproveita regras de alocação para calcular o custo total de propriedade e os custos unitários da tecnologia. Ele utiliza métricas de modelo, como OpEx Dados reais, CapEx Dados reais, Orçamento, Previsão e outras para representar fluxos de custos distintos ou perspectivas de análise.

**Métrica** - Uma métrica é um valor quantitativo usado para medir custo, consumo ou desempenho, dentro do modelo Padrão de Custeio. Métricas como horas OpEx,CapEx, de CPU, GBs de armazenamento, horas de trabalho, volumes de tickets ou contagens de usuários costumam servir como impulsionadores nas alocações e fornecem a base para calcular custos unitários e comparar o desempenho da tecnologia ao longo do tempo. As métricas do modelo são fundamentais para impulsionar os modelos reais; enquanto as métricas calculadas são predominantemente utilizadas para fins de relatórios e para definir valores ponderados nas alocações em todo o modelo.

**Alocação** - Uma alocação é uma regra de modelagem que define como os custos fluem ATUM, por exemplo, de mão de obra para torres de recursos ou de servidores para aplicativos. Ele utiliza chaves de alocação definidas tanto no objeto de envio quanto no de recebimento para estabelecer a conectividade entre os objetos e permite definir um determinado valor ponderado para orientar as alocações gerenciadas e consumidas. As alocações distribuem os custos com base em fatores como horas de trabalho, consumo, unidades ou valores de configuração.

**Personalização** - Uma personalização é qualquer alteração feita no modelo padrão de cálculo de custos, conjuntos de dados ou relatórios. As personalizações permitem que as organizações adaptem o Costing Standard ao seu ambiente, mas podem exigir atenção durante as atualizações.

**Atualização** - Uma atualização traz seu projeto ou componentes para a versão mais recente do Padrão de Custeio. Isso pode incluir alocações atualizadas, relatórios revisados, correções de bugs, novos recursos de componentes ou atualizações da taxonomia TBM.
