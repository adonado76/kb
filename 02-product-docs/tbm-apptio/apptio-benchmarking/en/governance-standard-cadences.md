# Standard cadences

You do not need daily anything. You need a few solid cadences.

## Annual

- Full benchmark refresh Move Benchmarking to the latest full fiscal year of your data By end of 2 nd quarter of each calendar year, verify latest benchmark datasets are loaded
- Peer group review Confirm industry, size band, region still make sense Adjust only if your business or IT scope has genuinely changed
- Scope and configuration review Validate Cost Pools, Towers, Sub-Towers mappings Document any changes that will affect trend comparability

## Quarterly

- Drift check Spot-check a handful of core metrics (Industry, OpEx, a couple infra) Look for unexplained jumps after model changes
- Governance usage Bring 2–3 benchmark views to key QBR / steering forums Focus on trends and deltas, not every metric in the catalog

## Monthly (minimal)

- Respond to ad hoc questions
- Capture new use cases and issues
- Keep small documentation updates in sync (for example config changes)
