## Integrated Investment Planning

O planejamento integrado das despesas do projeto elimina os silos de operações e investimentos para aumentar a precisão. Os principais benefícios incluem:

- Minimizar surpresas orçamentárias decorrentes dos custos de cauda longa do projeto (aumento dos custos operacionais e de suporte devido a projetos anteriores)
- Gerenciar recursos de forma mais eficaz em projetos e operações e garantir que os projetos financiados permaneçam dentro do orçamento
- Fornecer aos proprietários de recursos a transparência da demanda do projeto para melhorar a agilidade financeira

Para obter informações detalhadas sobre Integrated Investment Planning, consulte [Introdução ao Integrated Investment Planning](../planning/iip/gs-integrated-investment-planning.html).

## Automatizado Data Management (Beta)

Os usuários são habilitados com o compartilhamento automatizado de dados de referência do Cost Transparency para o Planning e a capacidade de importar dados planejados sob demanda do Cost Transparency para um plano específico.

Observe que esse recurso está atualmente em versão beta e disponível apenas para clientes selecionados identificados para o programa beta. Assim que concluirmos a fase beta, ela será implementada para todos os clientes. A menos que você seja um cliente beta, não será possível ativar ou usar esse recurso.

Com o Automated Data Management, os seguintes recursos são adicionados:

## Automatizado Data Management para dados de referência

Os clientes usam o site Cost Transparency Integration para importar dimensões de dados de referência e reais do Cost Transparency. Para obter mais informações, consulte [Trabalhando com o Cost Transparency Integration Panel](../planning/cti_panel.html "O painel atualizado da Integração da Transparência de Custos (CTI), introduzido na versão de planejamento 2.78, é um espaço independente em que os usuários administradores podem executar todas as tarefas da Integração da Transparência de Custos em um único painel de controle. O novo painel oferece gerenciamento centralizado e importação e exportação com um clique, com transparência de custos."). No entanto, a importação de dados do Cost Transparency é um processo manual.

Quando os dados de referência atualizados da Transparência de custos são necessários no planejamento, o administrador precisa saber que ocorreram mudanças nos dados de referência no lado da Transparência de custos, e há um tempo de espera para que os cálculos da Transparência de custos sejam concluídos. Portanto, a fila de cálculo deve ser monitorada ativamente para que seja concluída antes que os dados possam ser puxados para o planejamento. Quando os dados estiverem prontos, o administrador de planejamento deverá clicar manualmente nos botões da página Cost Transparency Integration no Planning e resolver quaisquer conflitos devido a dependências entre dimensões, conforme descrito no [Diagrama de Dependências da Tabela de Referência](https://community.ibm.com/community/user/viewdocument/reference-table-dependencies-diagra?CommunityKey=4100dfb8-fc23-4203-83c7-019253cf7c0b&tab=librarydocuments "(Abre em uma nova guia ou janela)"). A extração manual de dados de referência da Transparência de custos pode se tornar demorada e um processo doloroso para depurar e resolver os conflitos.

O recurso Automated Data Management resolve esse problema, permitindo que o usuário importe automaticamente os dados de referência do Cost Transparency assim que estiverem disponíveis. Não há mais necessidade de coordenar os dois sistemas para descobrir o momento certo de extrair os dados, e não há necessidade de clicar manualmente nos botões para importar dimensões, aliviando assim o usuário da dor de resolver os conflitos de dependência de dimensão.

## Automatizado Data Management para reais

A importação de dados reais é manual nesta versão beta. No entanto, a importação automatizada de dados reais estará disponível antes que o recurso Data Management automatizado esteja disponível de modo geral.

## Automatizado Data Management para dados de linha de base do plano

Os clientes trazem dados de linha de base para o planejamento, como o número de funcionários existentes, ativos fixos, contratos existentes, etc., de sistemas de fontes externas para o site TBM Studio no Cost Transparency. O recurso de modelagem fornecido pelo ApptioOne - Cost permite que eles façam mapeamentos e transformações de dados para que os dados importados de sistemas externos sejam consumíveis para o processo de planejamento do orçamento financeiro. No momento, os clientes precisam exportar manualmente esses dados de ApptioOne - Cost e, em seguida, importar manualmente para um plano específico em ApptioOne - Plan. No entanto, esse problema foi resolvido com o lançamento desse novo recurso.

Com o Automated Data Management for Plan Baseline Data, os clientes podem definir o mapeamento entre o conjunto de dados em TBM Studio e a tabela de despesas em ApptioOne - Plan usando o recurso Automated Data Management 's Entity Mapping e obter dados com o clique de um botão em um plano específico, conforme e quando necessário.

## Comparação de planos aprimorados

Os planos agora podem ser comparados alinhando os exercícios fiscais correspondentes ou alinhando o ano de início de ambos os planos.

## Como esse recurso pode ajudá-lo

Ao fazer a comparação de planos usando a opção Add Comparison (Adicionar comparação ) da tabela Expenses (Despesas) ou usando Compare Shortcuts (Atalhos de comparação ), os usuários precisam selecionar um método de alinhamento para comparar planos. Há dois métodos de alinhamento:

- Ano de início do plano - dois planos são comparados alinhando-se o ano de início de ambos os planos.

Por exemplo, se o Plano A for um plano de dois anos com os anos fiscais de 2019 e 2020, e o Plano B for um plano de dois anos com os anos fiscais de 2020 e 2021, esse método de comparação comparará o ano fiscal de 2019 do Plano A com o ano fiscal de 2020 do Plano B.

- Ano fiscal coincidente - dois planos são comparados alinhando os anos fiscais coincidentes entre os dois planos.

Por exemplo, se o Plano A for um plano de dois anos com os exercícios fiscais de 2019 e 2020, e o Plano B for um plano de dois anos com os exercícios fiscais de 2020 e 2021, esse método de comparação comparará o exercício fiscal de 2020 do Plano A com o exercício fiscal de 2020 do Plano B.

## Mais informações sobre esse recurso

Para obter mais informações, consulte [Comparar versões ou planos](../planning/compare-versions-plans.html)

## Integrated Investment Planning

Integrated Investment Planning permite que a liderança de TI obtenha uma visão aprofundada do custo real dos investimentos e do impacto no orçamento atual e futuro de TI, ao mesmo tempo em que acompanha os custos de investimentos individuais para planejar e fazer ajustes em tempo real.os investimentos podem ser um projeto, um portfólio épico ou um fluxo de valor, dependendo da terminologia ou do modelo de execução usado em sua organização. Os custos de investimento, por exemplo, mão de obra interna e externa, infraestrutura, fornecedor e outras despesas relacionadas, podem ser planejados a partir da perspectiva de construção e execução para fornecer um quadro completo do custo total de propriedade e, ao mesmo tempo, informar sobre as necessidades futuras do orçamento operacional.

Integrated Investment Planning o recurso pode ajudá-lo a realizar o seguinte:

- Experiência em planejamento integrado para orçamento de investimento e orçamento operacional.
- Capacidade de alocar recursos para um investimento planejado, usando métodos de alocação baseados em funções ou nomes.
- Suporte configurável para carga e carga cruzada.
- Analisar e relatar os custos de Run and Grow.
- Crie relatórios personalizados de despesas de investimento usando o site Apptio BI.
- Acompanhe os custos reais de investimento e compare-os com um plano.

Dica: Integrated Investment Planning é aplicável a clientes do Costing & Planning e do Costing & Planning Plus que não tenham o Project Financial Planning ou o Service Demand Planning ativado. Se você estiver usando Project Financial Planning ou Service Demand Planning e quiser acessar Integrated Investment Planning, entre em contato com o suporte Apptio.

## Mais informações sobre esse recurso

Para mais informações sobre Integrated Investment Planning, veja [Começar com Integrated Investment Planning](../planning/iip/gs-integrated-investment-planning.html)

## 3.12 - 24 de janeiro de 2022

A partir de segunda-feira, 24/01/2022, os principais locatários de produção começarão a fazer upgrade para a versão 3.12. Esta versão adiciona uma experiência aprimorada de filtros de itens de linha.

## Filtros de itens de linha aprimorados

Para dimensões que usam o código como um identificador exclusivo, por exemplo, departamento, conta, centro de custo ou projeto, os usuários agora podem definir filtros de itens de linha (também conhecidos como listas de opções dependentes) usando o código exclusivo.

## Novos filtros

Para definir um novo filtro com uma dimensão que usa Código como identificador exclusivo, ao carregar mapeamentos de valores para o filtro, você deve fornecer o valor do código em vez do valor do nome no arquivo.csv que será usado para carregar os dados de mapeamento de valores. A interface de usuário do filtro e a tabela de itens de linha de despesas exibem o nome correspondente ao código usado no upload do arquivo.csv.

## Filtros existentes

Se você já tiver filtros definidos com dimensões que usam Código como identificador exclusivo, se exportar dados de mapeamento de valores de filtro, o arquivo.csv exportado exibirá o código correspondente ao nome que você usou ao definir esses filtros.

Para fazer alterações nos mapeamentos de valores, você deve usar o código para dimensões que tenham o código como identificador exclusivo. Se você não vir o valor correto ao usar valores filtrados na tabela de itens de linha de despesas, verifique se a definição do filtro está usando o código correto correspondente ao valor esperado na tabela de itens de linha.
