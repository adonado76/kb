# Customize Top Use Cases content

On the Detailed Invoice report, click  Top Use Cases  in the upper-right
corner to display information about how to analyze the invoice. Two default use cases ship as part
of the application. You can modify the text in the default use cases and add up to three more use
cases. There can be a total of five use cases. The default use case content is stored in a table
called Use Case Reference. Maintain the use cases content in an Excel spreadsheet, upload the
spreadsheet to the application, and append the table to the Use Cases table.

Field descriptions

The descriptions for the fields in the Use Case Reference table follow. The table is located in
the group in Project Explorer.

- Title- The name of the use case.
- Description- A description of the use case that is displayed next to
  the name.
- Steps- Suggested steps to follow to analyze the invoice.
- ID- A number between 1 and 5 that determines the order of the use
  cases.  
  For example, if you enter the number 2, the use case will be the second one in
  the Top Detail Use Cases pop-up window.
- Report- The name of the report(s) where the content will be used. At
  this time, the only valid value is "Invoice."
