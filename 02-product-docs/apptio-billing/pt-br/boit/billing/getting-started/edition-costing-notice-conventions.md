# Avisos sobre edição e custos

O faturamento sempre depende de um projeto de cálculo de custos, mas este guia mantém a rotulagem simples, concentrando-se nas edições de faturamento:

- **O Billing Essentials** é executado dentro de um projeto **do Costing Essentials**.
- **O Padrão de Faturamento** funciona como um ponto final separado, além de um projeto **do Padrão de Custeio**.

Sempre que um tópico se aplica apenas a uma edição de Faturamento, um aviso aparece no topo da seção ou subseção:

- **Aviso: Aplica-se apenas ao Billing Essentials.**
- **Aviso: Aplica-se apenas ao padrão de faturamento.**

Se uma seção não exibir um desses avisos na parte superior, ela se aplica **tanto** ao Billing Essentials quanto ao Billing Standard.

Você pode assumir o seguinte:

- Qualquer conteúdo marcado como **Fundamentos de faturamento** inclui implicitamente o projeto Fundamentos de custos em que é executado.
- Qualquer conteúdo marcado como **Padrão de Faturamento** inclui implicitamente o projeto Padrão de Custeio e o ponto final em que é executado.

Dentro de uma seção, etapas ou tabelas específicas também podem ser identificadas como “Apenas essenciais para faturamento” ou “Apenas padrão para faturamento” quando o comportamento ou a configuração divergem.

## Convenções utilizadas neste guia

Para manter a consistência:

- **“Custeio”** significa o produto IBM Apptio anteriormente conhecido como Transparência de Custos.
- **“ IBMApptio Billing”** e **“Billing”** referem-se ao produto de faturamento geral (incluindo todas as edições).
- **“TBM Studio”** significa o ambiente de configuração e modelagem usado para criar e manter o conteúdo de Custos e Faturamento. Tarefas como carregamento de dados, definição de modelos, criação de relatórios e gerenciamento de compilações e ambientes são realizadas no TBM Studio por usuários com acesso apropriado.
- **“TBM”** significa a disciplina de Gestão de Negócios de Tecnologia e a taxonomia padrão para classificar os gastos com tecnologia.
