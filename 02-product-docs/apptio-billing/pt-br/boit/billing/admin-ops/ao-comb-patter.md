# Combinar padrões sem criar caos

A maioria das implementações reais acaba sendo uma mistura, por exemplo:

- Catálogo de serviços centrado em cobranças, com um pouco de marcação de projetos.
- Centrado na nuvem para domínios na nuvem e showback simples para todo o resto.
- Catálogo de serviços centrado e visualização da entidade jurídica apenas para um subconjunto de serviços.

Barreiras de proteção práticas:

- Decida qual padrão é o “primário” Esse padrão deve orientar a forma como você nomeia serviços, define unidades e estrutura relatórios.
- Use padrões adicionais para ampliar a visibilidade, não para criar esquemas de faturamento paralelos e concorrentes.
- Documente quais padrões estão ativos, para quais serviços e para quais públicos.

Se o modelo central for claro e documentado, esses padrões se tornam lentes poderosas sobre os mesmos dados, em vez de histórias separadas e conflitantes.
