# Design centrado no projeto

Observação: aplica-se apenas ao padrão de faturamento.

**Quando usar**

- Os gastos com projetos e iniciativas precisam ser visíveis em termos de faturamento.
- Os líderes querem ver quais projetos impulsionam o consumo e os custos de tecnologia.

**Orientar**

Duas variantes comuns:

1. **Projeto como consumidor faturado**
   - Os projetos recebem cobranças como qualquer outro consumidor.
   - Útil quando os custos de tecnologia precisam ser capitalizados ou alocados diretamente aos projetos.
2. **O projeto como lente de reportagem**
   - Os encargos continuam a ser atribuídos às unidades de negócio ou aos produtos.
   - Os atributos do projeto identificam os custos para apoiar a análise do portfólio.

**Elementos-chave**

- Dados mestre do projeto alinhados com PPM e Finanças.
- Mapeamento:
  - Mão de obra, infraestrutura e uso da nuvem para projetos.
  - Projetos para serviços ou aplicações, se relevante.
- Relatórios de faturamento que podem ser filtrados por projeto, serviço e consumidor.

**Relatórios típicos**

- Cobranças por projeto e serviço.
- Visão do projeto de plataformas ou serviços compartilhados consumidos.
- Visualizações do portfólio:
  - Principais projetos por gastos com tecnologia.
  - Despesas do projeto por patrocinador ou portfólio.

**Notas de implementação**

- Seja claro sobre se os projetos são verdadeiras metas de faturamento ou apenas etiquetas. Misturar os dois sem clareza cria confusão.
- Mantenha os IDs dos projetos estáveis entre sistemas e anos, sempre que possível.
