# Governança de configuração

As decisões de configuração afetam a confiança. Trate-as como mudanças reais, não como preferências casuais.

Práticas recomendadas:

- Mantenha um breve resumo da configuração para cada ambiente:
  - Projeto de cálculo de custos utilizado.
  - Versão TBM.
  - Grupo(s) de pares selecionado(s).
- Mantenha um registro simples das alterações nas configurações de benchmarking:
  - Data, alteração, motivo, aprovador.
- Defina regras de aprovação:
  - O líder da TBM deve aprovar as alterações na versão, no projeto de cálculo de custos principal ou no grupo de pares.
  - A TBMA pode ajustar mapeamentos e configurações menores com documentação e validação básica.

Se uma escolha de configuração pode alterar um slide executivo, ela merece uma linha no registro.
