# Pré-requisitos

- **Antes de começar**
  - A Avaliação Comparativa Interativa deve ser provisionada no Frontdoor para sua organização. Se você não tiver certeza ou não encontrar o botão “Interactive Benchmarking” (Benchmarking interativo) no seu ambiente Frontdoor, entre em contato com o seu ponto de contato IBM Apptio para obter ajuda.
  - Se você planeja integrar com o Cálculo de Custos, o projeto de Cálculo de Custos relevante deve existir e estar preenchido com pelo menos um ano fiscal completo de dados
- **Você precisa**
  - Acesso ao aplicativo **de Benchmarking Interativo**
  - Acesso ao projeto **de cálculo de custos** que alimentará relatórios com benchmark (se aplicável).

  Observação: se você não conseguir visualizar o Benchmarking ou não conseguir abrir o projeto de Cálculo de Custos relevante, não será possível avançar. O acesso precisa ser corrigido.
- **Dados mínimos para que valha a pena fazer isso**
  - Dados do perfil organizacional
    - Receita anual para o escopo da organização de TI que você está comparando
    - Contagem de funcionários ou FTE para o mesmo escopo.
    - Setor e região.
  - Dados de custos de TI
    - Gastos anuais com TI por grupo de custos da TBM.
    - Gastos anuais com TI pela TBM Resource Tower.
    - Opcional, mas recomendado: alocação de nível de sub-torre para infraestrutura.
  - Dados de volume de infraestrutura (somente se você planeja usar benchmarks de infraestrutura imediatamente)
    - Número de servidores, TB de armazenamento, dispositivos ou portas de rede e métricas semelhantes.
    - Idealmente, a contagem de ETIs para as principais funções de infraestrutura.

Lembre-se de que você pode começar com benchmarks de indústria e TI OpEx sem volumes de infraestrutura e adicionar a infraestrutura posteriormente.
