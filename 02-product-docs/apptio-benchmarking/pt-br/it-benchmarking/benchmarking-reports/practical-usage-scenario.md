# Cenários práticos de uso

Aqui estão algumas maneiras valiosas de usar os relatórios sem complicar demais as coisas.

**Defendendo um orçamento que parece “muito** alto” A   
situação..O departamento financeiro ou a diretoria considera que os gastos com TI são elevados e deseja reduzi-los.

Como usar os relatórios:

1. Comece com uma visão **dos gastos com TI** :
   - Despesas com TI como porcentagem das despesas da Resource Tower Pool de custos em comparação com os benchmarks
   - Mostre onde você se situa em relação ao seu arquétipo.

     ![Aba "Gastos com TI" do relatório "Referências de Benchmarks da Indústria e de Despesas OpEx " da Costing.](../../resources/images/it%20benchmarking_images/benchmark-it-spend-tab.png)
2. Passe para **a Distribuição de Aplicativos** :
   - Porcentagem de gastos com TI dividida por subtorres de recursos dentro da torre de recursos “Aplicativos” e comparada com benchmarks
   - Mostre onde você se situa em relação ao seu arquétipo.
3. Somente então, se necessário, acesse **Infraestrutura** e Cálculo de custos com benchmark ativado para torres específicas.

Objetivo: mudar a conversa de “reduzir em 10%” para “concordamos que este é o modelo operacional correto, considerando nossa estratégia e perfil de risco?”

**Visando uma iniciativa**  
 de otimização de custos A situação... você precisa encontrar economias confiáveis sem cortes aleatórios generalizados.

Como usar os relatórios:

1. Use as visualizações **de TI ( OpEx )** e **infraestrutura** para identificar torres ou pools de custos atípicos.
2. Para um outlier selecionado, abra um relatório **de custos habilitado para benchmark**.
3. Exercício para identificar os fatores determinantes:
   - Fornecedores específicos
   - Regiões ou centros de dados específicos
   - Serviços ou plataformas específicos.
4. Transforme isso em iniciativas:
   - Otimização de licenças
   - Racionalização e consolidação
   - Mudanças no abastecimento
   - Atualização ou classificação de hardware.

Objetivo: Vincule cada iniciativa a uma lacuna de referência visível; assim, fica muito mais fácil defendê-la e acompanhá-la.

**Planejamento** da transformação da torre  
 A situação... você planeja uma mudança plurianual para uma torre, como Usuário Final, Rede ou Armazenamento.

Como usar os relatórios:

1. Use benchmarks **de infraestrutura** para estabelecer uma linha de base para o custo unitário atual e a eficiência do FTE.
2. Defina uma **faixa-alvo** realista com base nos quartis, e não no mínimo absoluto.
3. Trabalhe com o proprietário da torre para identificar:
   - Que mudanças na arquitetura, ferramentas ou fornecimento o levariam a essa faixa?
   - Que investimento é necessário?
4. Acompanhe o progresso anualmente usando as mesmas métricas de referência.

Objetivo: usar referências para definir expectativas e mostrar progresso, não para buscar a perfeição.
