# Modelo operacional e funções

## Funções principais

Líder do Programa TBM/ITFM

- Entende como o Benchmarking se encaixa no programa TBM/ITFM
- Decide quando e onde os benchmarks são utilizados
- Aprova alterações importantes na configuração e no grupo de pares

Proprietário da avaliação comparativa

- Configura e mantém o Benchmarking
- **NOTA:** Consulte as seções **[Guia](../home.html#benchmarking__config_guide)** **[de início rápido](../home.html#benchmarking__quickstart)** e configuração para obter mais informações.
- Valida resultados e prepara análises
- Mantém a documentação, atualiza o calendário e o registro de alterações

Apptio Administrador da plataforma

- Gerencia o acesso aos aplicativos
- Garante que os ambientes e as conexões permaneçam saudáveis
- Abre e acompanha os tickets de suporte com Apptio quando necessário

Proprietários da Resource Tower / Líderes de domínio

- Consuma visualizações de referência relevantes para suas torres de recursos ou domínios
- Forneça contexto para as lacunas em relação aos colegas
- Ações conjuntas para abordar lacunas materiais

Finanças / FP&A

- Utiliza referências como base para definir metas e planejar
- Ajuda a alinhar as visões baseadas em referências com as visões financeiras internas
- Desafios, escopo e premissas, quando necessário

## RACI leve

O quadro RACI abaixo fornece orientações iniciais sobre as responsabilidades das principais funções e outros interesses. Se for utilizado, considere incorporar o quadro RACI em sua prática mais ampla de TBM/ITFM, juntamente com quaisquer detalhes específicos da empresa.

|  |  |  |  |  |  |
| --- | --- | --- | --- | --- | --- |
| **Atividade** | **Líder TBM** | **Linha de base para testes**  **Proprietário** | **Plataforma**  **Admin** | **Proprietário da Torre de Recursos** | **Financeiro** |
| Defina como a análise comparativa é utilizada | A/R | C | C | C | C |
| Configurar e manter o Benchmarking | C | R | C | I | I |
| Aprovar política de grupo de pares e escopo | A/R | C | I | C | C |
| Atualização anual do benchmark | A | R | C | C | C |

| Responda às perguntas de referência executiva | A/R | R | I | C | C |

*R = Responsável, A = Responsável, C = Consultado, I = Informado*
