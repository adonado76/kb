# Operating model and roles

## Key roles

TBM / ITFM Program Lead

- Owns how Benchmarking fits into the TBM / ITFM program
- Decides when and where benchmarks are used
- Approves major configuration and peer group changes

Benchmarking Owner

- Configures and maintains Benchmarking
- **NOTE:** See **[Quickstart](../home.html#benchmarking__quickstart)** &
  **[Configuration Guide](../home.html#benchmarking__config_guide)** sections for
  more information.
- Validates results and prepares analyses
- Maintains documentation, refresh calendar, and change log

Apptio Platform Admin

- Manages application access
- Ensures environments and connections stay healthy
- Opens and tracks support tickets with Apptio when needed

Resource Tower Owners / Domain Leads

- Consume relevant benchmark views for their Resource Towers or domains
- Provide context for gaps vs peers
- Co-own actions to address material gaps

Finance / FP&A

- Uses benchmarks as an input to target setting and planning
- Helps align benchmark-based views with internal financial views
- Challenges scope and assumptions where needed

## Lightweight RACI

Below RACI chart provides initial guidance of key roles’ responsibilities and other interests. If
used, consider incorporating the RACI chart into your larger TBM/ITFM practice along with any
company-specific details.

|  |  |  |  |  |  |
| --- | --- | --- | --- | --- | --- |
| **Activity** | **TBM Lead** | **Benchmarking**  **Owner** | **Platform**  **Admin** | **Resource Tower Owner** | **Finance** |
| Define how Benchmarking is used | A/R | C | C | C | C |
| Configure and maintain Benchmarking | C | R | C | I | I |
| Approve peer group and scope policy | A/R | C | I | C | C |
| Annual benchmark refresh | A | R | C | C | C |
| Use benchmarks in budget / planning | A/R | C | I | C | R |
| Use benchmarks in Resource Tower improvement programs | C | C | I | A/R | C |
| Respond to executive benchmark questions | A/R | R | I | C | C |

*R = Responsible, A = Accountable, C = Consulted, I = Informed*
