# Gobernanza de la configuración

Las decisiones de configuración afectan a la confianza. Trátalos como cambios reales, no como preferencias ocasionales.

Prácticas recomendadas:

- Mantenga un breve resumen de la configuración para cada entorno:
  - Coste del proyecto utilizado.
  - Versión TBM.
  - Grupo(s) de pares seleccionado(s).
- Mantenga un registro de cambios sencillo para la configuración de Benchmarking:
  - Fecha, cambio, motivo, aprobador.
- Definir reglas de aprobación:
  - El responsable de TBM debe aprobar los cambios en la versión, el proyecto principal de cálculo de costes o el grupo de pares.
  - TBMA puede ajustar las asignaciones y configuraciones menores con documentación y validación básica.

Si una opción de configuración puede cambiar una diapositiva ejecutiva, merece una línea en el registro.
