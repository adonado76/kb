# Entornos

TBM Studio utiliza tres entornos para facilitar tu flujo de trabajo de desarrollo.

## Entorno de Desarrollo

**Objetivo:** Tu espacio de trabajo personal para editar y realizar pruebas

**Puntos clave:**

- Activo cuando tienes documentos prestados
- Los cambios se aplican únicamente a tu espacio de trabajo
- Se muestran los entornos de desarrollo de otros usuarios si tienen documentos extraídos
- Puedes ver los entornos de desarrollo de otros usuarios si tienes los permisos necesarios

**Cuándo utilizarlo:** al realizar cambios en tablas, modelos o informes

## Entorno de pruebas

**Objetivo:** Entorno compartido en el que se recopilan todos los cambios realizados por los usuarios

**Puntos clave:**

- Activo cuando no hay ningún documento prestado
- Contiene todos los cambios registrados por todos los usuarios
- Se utiliza para realizar pruebas y validaciones antes de la producción
- Vista predeterminada para la mayor parte del trabajo de desarrollo

**Cuándo utilizarlo: para** revisar los cambios integrados de varios usuarios

## Entorno de producción

**Objetivo:** Entorno operativo para los usuarios finales y la elaboración de informes

**Puntos clave:**

- Accesible únicamente para los administradores con fines promocionales
- Acceso de solo lectura para los usuarios finales
- Contiene configuraciones estables y probadas
- Ascendido desde la fase de preparación por los administradores

**Cuándo utilizarlo:** para ver la versión actual en vivo disponible para los usuarios finales

**Para cambiar de entorno:** utiliza el menú desplegable **«Entorno»** de la barra de herramientas y selecciona «Desarrollo», «Preproducción» o «Producción».
