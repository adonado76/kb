# Build & Deployment Lifecycle

TBM Studio uses a controlled build and deployment lifecycle to ensure data integrity, enable
collaboration, and protect production environments from untested changes. Understanding this
lifecycle is essential for administrators who manage projects and for any user who makes
configuration changes.

This section explains how changes flow through TBM Studio's three environments, how the check-out
and check-in workflow protects data integrity, how builds calculate your model, and how changes are
promoted to production.
