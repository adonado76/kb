# Putting It All Together: Your First Report

Now that you understand the basics, here's a typical workflow for creating a report:

- [Create the report](report-basic.html) with a clear name and
  description
- [Add a table](add-tables-report.html) to show detailed data—choose the
  right model object, add dimensions to Rows, add metrics to Values
- [Add a chart](add-chart-visual.html) to visualize key patterns—choose
  the right chart type for your message
- [Add slicers](use-slicers.html) for interactivity—include slicers for
  dimensions users commonly filter by
- [Review and check in](others-report.html)—verify the report displays
  correctly, test slicer interactions, check in to save your changes

## Quick Reference: Report Components

|  |  |  |  |
| --- | --- | --- | --- |
| **Component** | **Add From** | **Configuration Areas** | **Best For** |
| Table | Report tab > Table | Rows, Columns, Values, Filters | Detailed data, exports |
| Chart | Report tab > Chart | Legend, Axis, Values, Filters | Visual patterns, comparisons |
| Row Slicer | Report tab > Row Slicer | Slice By, Filters | Interactive filtering |
| Compact Slicer | Report tab > Compact Slicer | Slice By (multiple) | Space-efficient filtering |
