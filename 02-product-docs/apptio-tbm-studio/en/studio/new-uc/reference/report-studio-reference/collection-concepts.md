# Collection Concepts

**What is a Report Collection**

A report collection is a named container that groups related reports together. When reports
belong to the same collection, users can easily navigate between them using a report collection
navigator component that appears in each report. Collections provide a cohesive viewing experience
for reports that share a common purpose, such as monthly financial reviews, executive dashboards, or
departmental cost analyses.

Key characteristics of report collections:

- Each report can belong to only one collection at a time
- Collections are available across an entire project once created
- Collections support role-based access control
- Color palettes can be applied at the collection level for consistent branding

**Collection vs. Folder Organization**

Report collections differ from folder-based organization in the Project Explorer. Folders in the
Project Explorer help content creators organize reports during development, while collections are
designed for end-user navigation and consumption.

|  |  |  |
| --- | --- | --- |
| **Feature** | **Report Collections** | **Project Explorer Folders** |
| Purpose | End-user navigation | Content organization during development |
| Visibility | Visible to report consumers | Visible to content creators |
| Navigation | Built-in navigator component | Manual navigation via Project Explorer |
| Permissions | Role-based access control | Follows report-level permissions |
| Styling | Supports collection-level color palettes | No styling features |

**Use Cases for Collections**

Report collections are particularly valuable for the following scenarios:

**Monthly Reporting Packages**

- Group all reports needed for a monthly financial review
- Include cost summary, variance analysis, and trend reports in a single collection

**Executive Dashboard Suites**

- Combine high-level KPI dashboards with drill-down detail reports
- Provide executives with a cohesive view of IT financial health

**Departmental Views**

- Create collections for each business unit or cost center
- Allow department heads to navigate their relevant reports easily

**Topic-Based Organization**

- Group reports by subject matter (e.g., Infrastructure Costs, Application Costs, Labor Costs)
- Help users find reports related to specific TBM taxonomy categories
