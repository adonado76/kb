# Tables in reports

**Applies to**: TBM Studio 12.0 and later

Tables are a useful way to present data and are a common element in application reports. The
basic types of tables are described below.

## Sample tables

A report with several sample tables is shown in the following image. All the tables present data
about applications. The table on the left is object-based. The table on the right is a transform
table.

![image](../../../../../../03-media/apptio-tbm-studio/resources/images/studio_images/studioimages/studio/stu611.png)

## Modify tables

After you add a table to a report, you can modify it by selecting the table and clicking the
**Ad Hoc** tab. The **Ad Hoc** tab is displayed only when you select an object-based or
transform table. The **Ad Hoc** tab is shown in the following image.

![image](../../../../../../03-media/apptio-tbm-studio/resources/images/studio_images/studioimages/studio/stu612.png)

## Analyze tables

You can analyze the content in table columns by right-clicking the header of a column, and then
clicking **Show Values**. A table is displayed that shows each value and the number of times the
value occurs in the column.

## Legacy tables

Prior to the addition of the Ribbon to Report Studio, tables were created and edited using a wide
variety of tools. The tables often were modified and formatted by editing the data path for a chart.
Legacy tables can be modified using many of the tools in the Ribbon. However, not all tools
available to ad-hoc tables are available to legacy tables.

You can convert an ad hoc table to a legacy table. However, once you convert the table, you
cannot convert it back to an ad hoc table. To convert a table, open the **Table** menu in the
upper-left corner of the table frame and select **Convert to Legacy Table**.
