# Layout do relatório

**Aplica-se a** : TBM Studio 12.0 e posterior

Depois de adicionar componentes a um relatório, você pode organizá-los usando as ferramentas de layout do relatório na guia **Format**.

## Guia Formatar

A guia **Format** inclui ferramentas para alinhar componentes, distribuí-los uniformemente na horizontal e na vertical e uniformizar o tamanho dos componentes.

## Caixas de grupo

As caixas de grupo são adições cosméticas aos relatórios e são úteis para agrupar visualmente os componentes em um relatório. Eles também agrupam fisicamente os componentes para que possam ser movidos como um grupo. Para adicionar uma caixa de agrupamento, clique em **Group (Grupo** ) na guia **Report (Relatório** ).

## Componentes com guias

Para que os usuários tenham acesso rápido a uma série de componentes do relatório sem precisar rolar a tela, é possível colocar os componentes em um grupo com guias. Todos os componentes, exceto as caixas de agrupamento, podem ser colocados em uma guia. Para adicionar um grupo com guias, clique em **Tabs (Guias** ) na guia **Report (Relatório** ).

## Dimensionamento e posicionamento dinâmicos

O dimensionamento e o posicionamento dinâmicos ajustam os componentes com base no conteúdo de tabelas, gráficos, caixas de grupo e componentes com guias. Para usar o dimensionamento dinâmico, clique em **Dynamic (Dinâmico** ) na guia **Format (Formato** ).

## Relatórios mestre

Para garantir que um conjunto de relatórios tenha o mesmo layout, você pode criar um relatório mestre e aplicá-lo ao conjunto de relatórios. Um relatório mestre pode incluir todos os componentes de um relatório normal, como botões e caixas de grupo. Para aplicar um relatório mestre, clique em **Mestres** na guia **Relatório**.

## Relatar projetos

Alguns aplicativos vêm com relatórios de projeto. Um relatório de blueprint é um modelo de layout baseado em um relatório padrão que acompanha o aplicativo. Se quiser criar um relatório personalizado que tenha o mesmo layout de um dos relatórios padrão, você poderá usar um blueprint. Para obter mais informações, consulte [Adicionar componentes de um blueprint](add-components-from-blueprint.html).
