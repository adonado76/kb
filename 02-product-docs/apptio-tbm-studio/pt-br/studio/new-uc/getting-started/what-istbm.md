# O que é o TBM Studio?

|  |  |
| --- | --- |
| **Tipo de Conteúdo** | Conceitual |

| **Tempo de conclusão** | 3 a 5 minutos |

## Introdução

O TBM Studio é uma plataforma abrangente que ajuda as organizações a compreender, gerenciar e otimizar seus gastos com tecnologia. Seja para monitorar custos de TI, alocar despesas às unidades de negócios ou criar painéis executivos, o TBM Studio oferece as ferramentas necessárias para transformar dados financeiros e operacionais brutos em insights úteis.

## O que a TBM Studio faz

O TBM Studio resolve um desafio crucial que a maioria das equipes de TI e finanças enfrenta: como acompanhar com precisão os custos de tecnologia e demonstrar o valor que a TI agrega aos negócios. Sem as ferramentas certas, os dados de custos ficam dispersos em planilhas isoladas; as alocações são feitas manualmente e estão sujeitas a erros, e a liderança não tem visibilidade sobre para onde estão indo os recursos destinados à tecnologia.

O TBM Studio enfrenta esses desafios oferecendo:

- **Gerenciamento unificado de dados** : Reúna dados de custos provenientes de livros contábeis, faturas de fornecedores, contas de serviços em nuvem e outras fontes em uma única plataforma
- **Modelagem flexível de custos** : alocar os custos de tecnologia desde sua origem, passando pelas camadas de infraestrutura, até as aplicações e serviços que os utilizam
- **Relatórios interativos** : crie painéis e relatórios que permitam às partes interessadas analisar custos, identificar tendências e tomar decisões baseadas em dados

O resultado é uma visão clara e fundamentada dos custos de tecnologia, que auxilia no orçamento, no repasse de custos, na otimização de custos e no planejamento estratégico.

## Os três componentes principais

O TBM Studio é composto por três estúdios integrados que trabalham em conjunto para dar suporte a todo o fluxo de trabalho, desde os dados até as conclusões:

## Data Studio : Prepare seus dados

**Objetivo** : coletar, transformar e aprimorar seus dados para que estejam prontos para análise.

Data Studio é onde você importa os dados brutos para o TBM Studio e os adapta de acordo com suas necessidades. Será possível:

- Carregar dados de arquivos do Excel, do CSV ou de outras fontes
- Transforme os dados por meio de um fluxo visual de operações (filtragem, junção, pivô, etc.)
- Enriqueça os dados com fórmulas e colunas calculadas
- Gerenciar versões mensais dos dados para acompanhar as alterações ao longo do tempo
- Divida os dados por data para distribuir automaticamente as informações por períodos de tempo

**Quando usar** : Quando precisar preparar dados de custos a partir do seu razão geral, mapear nomes de fornecedores para uma taxonomia padrão ou combinar várias fontes de dados em uma única visualização.

**Exemplo** : Sua organização recebe faturas mensais separadas de três provedores de serviços em nuvem. No Data Studio, você carrega cada fatura, padroniza os nomes dos fornecedores e as categorias de serviços e os agrupa em uma única tabela de “Serviços em nuvem” que mostra todos os gastos com nuvem em um formato consistente.

## Model Studio: Análise dos custos em toda a sua organização

**Objetivo** : Visualizar como os custos fluem desde sua origem até os serviços e unidades de negócios que os consomem.

O Model Studio é onde você cria modelos de alocação que rastreiam os custos de tecnologia em toda a sua organização. Será possível:

- Adicione tabelas de dados como objetos no seu modelo de custos
- Defina os fatores (como número de servidores, número de funcionários em tempo integral ou volume de transações) que determinam como os custos são alocados
- Crie alocações que distribuam os custos de um objeto para outro com base nesses fatores determinantes
- Visualize fluxos de custos usando diagramas de Sankey interativos
- Acompanhe várias métricas (custo, orçamento, previsão) por meio da mesma estrutura de modelo

**Quando usar** : Quando precisar distribuir custos compartilhados, realizar análises de transparência de custos ou dar suporte a iniciativas de chargeback e showback.

**Exemplo** : Seus custos de operações de TI começam com o objeto “Fonte de custo”, que representa seu razão geral. Você aloca esses custos à categoria “Serviços de Tecnologia” (como hospedagem na nuvem, serviços de banco de dados e e-mail) com base no consumo de recursos. Em seguida, você aloca recursos dos serviços de tecnologia para a categoria “Aplicativos”, com base em quais aplicativos utilizam quais serviços. Por fim, você aloca recursos das aplicações para as “Unidades de Negócio” com base no uso das aplicações, proporcionando a cada unidade de negócio uma visão clara de sua parcela nos custos de TI.

## Report Studio: Divulgar insights

**Objetivo** : Criar relatórios e painéis interativos que tornem seus dados acessíveis às partes interessadas.

O Report Studio é onde você cria os relatórios com os quais os usuários finais interagem. Será possível:

- Crie tabelas, gráficos e KPIs a partir dos seus dados modelados
- Adicione filtros e segmentadores para que os usuários possam explorar os dados de forma interativa
- Crie recursos de detalhamento para navegar do resumo aos detalhes
- Crie painéis específicos para cada função, voltados para diferentes públicos
- Agendar relatórios para distribuição automática

**Quando usar** : Quando precisar apresentar análises de custos aos executivos, disponibilizar relatórios de autoatendimento aos gerentes de unidades de negócios ou criar relatórios financeiros mensais.

**Exemplo** : Você cria um painel executivo que mostra os custos totais de TI por trimestre, com tendências ano a ano. O painel inclui filtros para unidades de negócios e categorias de custos. Quando um executivo clica em um pico de custos, ele pode aprofundar a análise para ver quais serviços específicos causaram o aumento e quais fornecedores estiveram envolvidos.

## Quem usa o TBM Studio e por quê

Diferentes funções utilizam o TBM Studio para atingir diferentes objetivos:

**As equipes de finanças de TI** utilizam o TBM Studio para:

- Acompanhar e relatar os gastos com TI em toda a organização
- Criar modelos de alocação para chargeback e showback
- Apoiar os processos de elaboração de orçamentos e previsões
- Demonstrar o valor da TI para a empresa

**Os analistas de negócios** utilizam o TBM Studio para:

- Analisar tendências de custos e identificar oportunidades de otimização
- Crie relatórios ad hoc para responder a questões comerciais específicas
- Manter a qualidade dos dados e resolver discrepâncias
- Oferecer análises em regime de autoatendimento às partes interessadas

**Os administradores** utilizam o TBM Studio para:

- Gerenciar o acesso e a segurança dos usuários
- Monitorar a atualidade e a qualidade dos dados
- Supervisionar o ciclo de vida da compilação e da implantação
- Manter dados de referência e configurações do projeto

**Os desenvolvedores** utilizam o TBM Studio para:

- Amplie a funcionalidade por meio de APIs e scripts
- Automatizar fluxos de trabalho de integração de dados
- Crie cálculos e transformações personalizados
- Integrar o TBM Studio com outros sistemas corporativos

## Como tudo funciona em conjunto

Aqui está um fluxo de trabalho típico que ilustra como os três estúdios trabalham em conjunto:

***Processo de encerramento do mês*** :

1. **Data Studio** : O departamento financeiro carrega o arquivo do razão geral do mês atual. O fluxo de transformação categoriza automaticamente as despesas, mapeia os nomes dos fornecedores e distribui os custos ao longo do mês fiscal.
2. **Model Studio** : A geração mensal é executada automaticamente, processando os novos dados de custos no modelo de alocação. Os custos dos servidores são alocados às aplicações com base no uso da CPU; os custos das aplicações são alocados às unidades de negócios com base no volume de transações.
3. **Report Studio** : Os usuários finais abrem seus painéis mensais e veem imediatamente os valores atualizados dos custos. Os gerentes das unidades de negócios analisam os custos que lhes foram atribuídos, identificam os desvios em relação ao orçamento e investigam detalhadamente os gastos incomuns.

Esse fluxo automatizado economiza horas de trabalho manual todos os meses, garantindo que as partes interessadas tenham sempre acesso a informações atualizadas e precisas sobre custos.

## Exemplo prático: Gestão de custos na nuvem

**O desafio** : Uma empresa de serviços financeiros de médio porte gasta anualmente $2M em infraestrutura em nuvem nos sites AWS, Azure e Google Cloud. Os custos estão aumentando 20% ano a ano, mas a equipe de TI não consegue explicar quais iniciativas de negócios estão causando esse aumento. As unidades de negócios reclamam que os custos da nuvem são uma “caixa preta”

**A solução com o TBM Studio** :

1. **Data Studio** : Carregue as faturas mensais de nuvem dos três provedores. Transforme os dados para padronizar os nomes dos serviços (por exemplo, “ ““EC2 ”, “Virtual Machines” e “Compute Engine” são todos mapeados para “Cloud Compute”). Combine com metadados internos para classificar os recursos por aplicativo e unidade de negócios.
2. **Model Studio** : Crie um fluxo de custos de provedores de nuvem → tipos de serviços em nuvem → aplicativos → unidades de negócios. Use tags de recursos como fatores de alocação. O modelo mostra exatamente quanto cada unidade de negócios gasta com computação, armazenamento e rede em todos os provedores de nuvem.
3. **Report Studio** : Crie painéis para três públicos-alvo:
   - **Executivos** : KPIs de alto nível que mostram o gasto total com nuvem, tendências de crescimento e custo por unidade de negócios
   - **Gerentes de Unidade de Negócios** : Visão detalhada dos custos de nuvem por tipo de aplicativo e serviço, com comparações mês a mês
   - **Operações de TI** : Visão técnica que mostra os custos por recurso, região e oportunidades de otimização

**Resultado** : a empresa obtém visibilidade total dos gastos com a nuvem. Eles constatam que 40% dos custos de armazenamento provêm de instantâneos de desenvolvimento antigos que podem ser excluídos com segurança. As unidades de negócios agora podem monitorar seu consumo de nuvem e são incentivadas a otimizá-lo. A equipe de TI pode demonstrar o valor que está agregando e justificar os pedidos de orçamento com base em dados.
