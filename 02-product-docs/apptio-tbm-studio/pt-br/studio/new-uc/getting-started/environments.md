# Ambientes

O TBM Studio utiliza três ambientes para dar suporte ao seu fluxo de trabalho de desenvolvimento.

## Ambiente de desenvolvimento

**Objetivo:** Seu espaço de trabalho pessoal para edição e testes

**Pontos principais:**

- Ativo quando você tem documentos retirados
- As alterações são específicas do seu espaço de trabalho
- Os ambientes de desenvolvimento de outros usuários são listados caso tenham documentos retirados
- Você pode visualizar os ambientes de desenvolvimento de outros usuários, desde que tenha as permissões necessárias

**Quando usar:** Ao fazer alterações em tabelas, modelos ou relatórios

## Ambiente de teste

**Objetivo:** Ambiente compartilhado onde todas as alterações feitas pelos usuários são agregadas

**Pontos principais:**

- Ativo quando você não tem nenhum documento retirado
- Contém todas as alterações registradas por todos os usuários
- Utilizado para testes e validação antes da produção
- Visualização padrão para a maior parte do trabalho de desenvolvimento

**Quando usar: para** revisar alterações integradas feitas por vários usuários

## Ambiente de Produção

**Objetivo:** Ambiente de produção para usuários finais e geração de relatórios

**Pontos principais:**

- Acessível apenas aos administradores para fins de divulgação
- Acesso somente para visualização para usuários finais
- Contém configurações estáveis e testadas
- Promovido da fase de teste pelos administradores

**Quando usar: para** visualizar a versão ativa atual disponível para os usuários finais

**Para alternar entre ambientes:** use o menu suspenso **“Ambiente”** na barra de ferramentas e selecione “Desenvolvimento”, “Preprodução” ou “Produção”.
