# Arquitetura de dados

Esta seção explica como os dados circulam pelo TBM Studio, desde a ingestão inicial até a geração dos relatórios finais. O objetivo é ajudá-lo a construir um modelo mental claro sobre onde seus dados estão armazenados, como são transformados e como a arquitetura da plataforma permite uma análise de custos precisa e oportuna.

Compreender a arquitetura de dados é essencial, independentemente da sua função. Os analistas de negócios se beneficiam ao compreender como os dados de fontes anteriores moldam os relatórios que eles utilizam. Os administradores precisam compreender como a atualidade e a validação dos dados afetam a precisão do modelo. Os desenvolvedores que trabalham com integrações precisam saber por onde os dados entram e saem da plataforma.
