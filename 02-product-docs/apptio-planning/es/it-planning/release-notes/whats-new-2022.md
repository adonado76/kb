## Integrated Investment Planning

La planificación integrada de gastos de proyectos elimina los silos de operaciones e inversiones para aumentar la precisión. Las principales ventajas son:

- Minimizar las sorpresas presupuestarias derivadas de los costes a largo plazo de los proyectos (aumento de los costes operativos y de apoyo debido a proyectos anteriores)
- Gestionar los recursos de forma más eficaz en todos los proyectos y operaciones, y garantizar que los proyectos financiados se mantienen dentro del presupuesto
- Proporcionar a los propietarios de recursos transparencia en la demanda de proyectos para mejorar la agilidad financiera

Para obtener información detallada sobre Integrated Investment Planning, consulte [Introducción a Integrated Investment Planning](../planning/iip/gs-integrated-investment-planning.html).

## Automatizado Data Management (Beta)

Los usuarios pueden compartir automáticamente datos de referencia de Transparencia de Costes a Planificación e importar datos del plan a petición desde Transparencia de Costes a un plan específico.

Tenga en cuenta que esta función se encuentra actualmente en versión beta y sólo está disponible para clientes seleccionados que se hayan identificado para el programa beta. Una vez finalizada la fase beta, se extenderá a todos los clientes. A menos que sea un cliente beta, no podrá activar ni utilizar esta función.

Con Automated Data Management, se añaden las siguientes capacidades:

## Automatización Data Management para datos de referencia

Los clientes utilizan Cost Transparency Integration para importar dimensiones de datos de referencia y reales de Transparencia de costes. Para más información, consulte [Trabajar con Cost Transparency Integration Panel](../planning/cti_panel.html "El panel actualizado de Integración de Transparencia de Costes (CTI), introducido en la versión de planificación 2.78, es un espacio autónomo en el que los usuarios administradores pueden realizar todas las tareas de Integración de Transparencia de Costes dentro de un único panel de control. El nuevo panel ofrece gestión centralizada e importación y exportación con un solo clic y Transparencia de costes."). Sin embargo, la importación de datos desde Transparencia de Costes es un proceso manual.

Cuando se requieren datos de referencia actualizados de Transparencia de Costes en la planificación, el administrador necesita saber que se han producido cambios en los datos de referencia en el lado de Transparencia de Costes, y hay un tiempo de espera para que se completen los cálculos de Transparencia de Costes. Por lo tanto, la cola de cálculo debe supervisarse activamente para que finalice antes de que los datos puedan extraerse a la planificación. Una vez que los datos están listos, el administrador de la planificación tiene que hacer clic manualmente en los botones de la página Cost Transparency Integration en Planificación y resolver cualquier conflicto debido a las dependencias entre dimensiones, como se describe en el [Diagrama de Dependencias de la Tabla de Referencia](https://community.ibm.com/community/user/viewdocument/reference-table-dependencies-diagra?CommunityKey=4100dfb8-fc23-4203-83c7-019253cf7c0b&tab=librarydocuments "(se abre en una pestaña o una ventana nueva)"). Extraer manualmente los datos de referencia de Transparencia de costes puede llevar mucho tiempo y convertirse en un proceso doloroso para depurar y resolver los conflictos.

La función automatizada Data Management resuelve este problema permitiendo al usuario importar automáticamente los datos de referencia de Transparencia de costes en cuanto están disponibles. Ya no es necesario coordinar los dos sistemas para averiguar el momento adecuado para extraer los datos, y no es necesario pulsar manualmente los botones para importar dimensiones, lo que libera al usuario de la pesada tarea de resolver los conflictos de dependencia de las dimensiones.

## Automatización Data Management para los datos reales

La importación de datos reales es manual en esta versión beta. No obstante, la importación automatizada de datos reales estará disponible antes de que se generalice la función Data Management.

## Automatizado Data Management para datos de referencia del plan

Los clientes aportan datos de referencia para la planificación, como el recuento de personal existente, los activos fijos, los contratos existentes, etc., desde sistemas de fuentes externas a TBM Studio en Transparencia de costes. La capacidad de modelización que ofrece ApptioOne - Cost les permite realizar mapeos y transformaciones de datos para que los datos importados de sistemas externos sean consumibles para el proceso de planificación del presupuesto financiero. Actualmente, los clientes tienen que exportar manualmente estos datos desde ApptioOne - Coste y luego importarlos manualmente a un plan específico en ApptioOne - Plan. Sin embargo, este problema se resuelve con el lanzamiento de esta nueva función.

Con Automated Data Management for Plan Baseline Data, los clientes pueden definir la correspondencia entre el conjunto de datos de TBM Studio y la tabla de gastos de ApptioOne - Plan mediante la función de correspondencia de entidades de Automated Data Management y obtener los datos con sólo pulsar un botón desde un plan específico cuando sea necesario.

A partir del lunes 30/05/2022, los principales inquilinos de producción comenzarán a actualizarse a la versión 3.21. Esta versión menor añade la nueva función de comparación de planes mejorada.

## Comparación del Plan Mejorado

Ahora los planes pueden compararse haciendo coincidir los ejercicios fiscales o haciendo coincidir el año de inicio de ambos planes.

## Cómo puede ayudarle esta función

Al realizar la comparación de planes mediante la opción Añadir comparación de la tabla de Gastos, o utilizando los Atajos de comparación, los usuarios deben seleccionar un método de alineación para comparar planes. Existen dos métodos de alineación:

- Año de inicio del plan : se comparan dos planes alineando el año de inicio de ambos.

Por ejemplo, si el Plan A es un plan de dos años con ejercicios fiscales 2019 y 2020, y el Plan B es un plan de dos años con ejercicios fiscales 2020 y 2021, este método de comparación comparará el ejercicio fiscal 2019 del Plan A con el ejercicio fiscal 2020 del Plan B.

- Ejercicio fiscal coincidente : se comparan dos planes alineando los ejercicios fiscales coincidentes entre dos planes.

Por ejemplo, si el Plan A es un plan de dos años con ejercicios fiscales 2019 y 2020, y el Plan B es un plan de dos años con ejercicios fiscales 2020 y 2021, este método de comparación comparará el ejercicio fiscal 2020 del Plan A con el ejercicio fiscal 2020 del Plan B.

## Más información sobre esta función

Para más información, consulte [Comparar versiones o planes](../planning/compare-versions-plans.html)

## Integrated Investment Planning

Integrated Investment Planning permite a los responsables de TI conocer en profundidad el coste real de las inversiones y su repercusión en el presupuesto de TI actual y futuro, al tiempo que realiza un seguimiento de los costes de inversión individuales para planificar y realizar ajustes en tiempo real.las inversiones pueden ser un proyecto, una épica de cartera o un flujo de valor en función de la terminología o el modelo de ejecución utilizado en su organización. Los costes de inversión, por ejemplo la mano de obra interna y externa, la infraestructura, los proveedores y otros gastos relacionados, pueden planificarse tanto desde la perspectiva de la construcción como de la ejecución para ofrecer una imagen completa del coste total de propiedad, al tiempo que se informa sobre las futuras necesidades presupuestarias operativas.

Integrated Investment Planning puede ayudarle a conseguir lo siguiente:

- Experiencia en planificación integrada del presupuesto de inversión y del presupuesto operativo.
- Capacidad para asignar recursos a una inversión planificada, utilizando métodos de asignación basados en funciones o en nombres.
- Carga configurable y soporte de carga cruzada.
- Analizar e informar de los costes de explotación y crecimiento.
- Cree informes de gastos de inversión personalizados utilizando Apptio BI.
- Haga un seguimiento de los costes de inversión reales y compárelos con un plan.

Consejo: Integrated Investment Planning es aplicable a los clientes de Costing & Planning y Costing & Planning Plus que no tengan habilitados Project Financial Planning o Service Demand Planning . Si utiliza Project Financial Planning o Service Demand Planning y desea acceder a Integrated Investment Planning, póngase en contacto con el servicio de asistencia Apptio.

## Más información sobre esta función

Para obtener más información sobre Integrated Investment Planning, ver [Comenzar con Integrated Investment Planning](../planning/iip/gs-integrated-investment-planning.html)

## 3.12 - 24 de enero de 2022

A partir del lunes 24/01/2022, los principales inquilinos de producción comenzarán a actualizarse a la versión 3.12. Esta versión añade una experiencia mejorada de los filtros de partidas.

## Filtros de partidas mejorados

Para las dimensiones que utilizan el Código como identificador único, por ejemplo Departamento, Cuenta, Centro de Coste o Proyecto, los usuarios pueden ahora definir Filtros de Partidas (también conocidos como Listas de Selección Dependientes) utilizando el código único.

## Filtros nuevos

Para definir un nuevo filtro con una dimensión que utilice Código como identificador único, al cargar las asignaciones de valores para el filtro deberá proporcionar el valor del código en lugar del valor del nombre en el archivo.csv que se utilizará para cargar los datos de asignación de valores. La interfaz de usuario del filtro y la tabla de partidas de gastos muestran el nombre correspondiente al código utilizado en la carga.csv.

## Filtros existentes

Si ya tiene filtros definidos con dimensiones que utilizan Código como identificador único, si exporta datos de asignación de valores de filtro, el archivo.csv exportado muestra el código correspondiente al nombre que utilizó al definir estos filtros.

Para realizar cambios en las asignaciones de valores, debe utilizar el código para las dimensiones que tienen código como identificador único. Si no ve el valor correcto al utilizar valores filtrados en la tabla de partidas de gastos, compruebe que la definición del filtro utiliza el código correcto correspondiente al valor que espera en la tabla de partidas.
